Para trabajar con docker compose hay que crear un archivo llamado docker-compose.yml, preferentemente en la raíz del proyecto

version: <"numero"> --> 

services: --> seguido de los servicios van los nombres que les queremos asignar a los contenedores

build: <ruta> --> va después del nombre de algunos contenedores e indica la ruta en la que se encuentra el dockerfile, si es en el directorio raíz va un punto

ports: /salto de línea /tabulación - <"puerto_anfitrión>:<puerto_contenedor"> --> va después del nombre de los contenedores y es para los puertos del anfitrión y del contenedor respectivamente

links: /salto de línea /tabulación - <contenedor> --> es para vincular el contenedor a otro contendor de la red

image: <imagen> --> va después del nombre de el contenedor de, por ejemplo, una base de datos, es la imagen a partir de la cual se construye un contenedor, generalmente, de base de datos

environment: /salto de línea /tabulación - <variable>=<valor> --> va después del nombre de un contenedor, sirve para configurar las variables de entorno

volumes: /salto de línea /tabulación <nombre> --> va al mismo nivel que servicios. Es para trabajar con docker volumes

volumes: /salto de línea /tabulación - <nombre>:<ruta> --> trabaja en conjunto con el anterior. Va después del contenedor que corresponda, generalmente de una base de datos. El nombre es el mismo que en el del volumes anterior. La ruta es adonde ser guarda la información dentro del contenedor. La ruta para mongoDB es /data/db, para MySQL es /var/lib/mysql, para postgres es /var/lib/postgresql/data
