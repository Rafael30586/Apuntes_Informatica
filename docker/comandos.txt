docker images --> muestra la lista de las imágenes

dokcer pull <imagen>:<versión> --> descarga la imagen de la versión que indiquemos. Sin no aclaramos la versión descargará la última

docker image rm <imagen>:<versión> --> elimina la imagen que indiquemos

docker create <imagen>:<versión> --> crea un contenedor a partir de la imagen que indiquemos

docker create --name <nombre> <imagen>:<versión> --> crea un contenedor con el nombre que indiquemos

docker start <id_o_nombre_de contenedor> --> ejecuta el contenedor según el id que indiquemos

docker ps --> muestra todos los contenedores en ejecución

docker stop <id_de_contenedor> --> detiene el contenedor según el id que indiquemos

docker ps -a --> muestra todos los contenedores, incluidos los que no están en ejecución

docker rm <id_o_nombre_de contenedor> --> borra el contenedor según el nombre o id que indiquemos 

docker create -p<puerto_anfitrión>:<puerto_contenedor> <imagen> --> crea una contenedor de la imagen que indiquemos haciendo un mapeo con el puerto de la máquina anfitrión y el del contenedor según indiquemos. Si colocamos solo un puerto en lugar de dos, docker asignará un puerto del anfitrión al azar

docker logs <contenedor> --> muestra el registro del contenedor que indiquemos

docker logs --folow <contenedor> --> muestra los registros de un contenedor y permanece a la escucha de los mismos

docker run <imagen> --> descarga la imagen que indiquemos (si es que no la tenemos descargada ya), crea un contenedor a partir de la imagen y lo ejecuta. Se le puede agregar la opción --name <nombre> para darle nombre al contenedor. Agregando la opción -d hace que el contenedor se ejecute en segundo plano

Cuando creamos un contenedor con el comando run y debemos utilizar muchas opciones como -d, -p, -e , etc, podemos separar a las opciones usando una barra "\" después de cada opción y hacer un salto de línea para que todo sea más legible y no esté en una sola línea, Ejemplo:
docker run -d \
-p 27017:27017 \
-e MONGO_INITDB_ROOT_USERNAME=admin \
-e MONGO_INITDB_ROOT_PASSWORD=password \
--name mobgodb \
--net mongo-network \
mongo   

docker <comando> --help --> muestra ayuda del comando que le indiquemos 

docker exec --> sirve para ejecutar comandos en un contenedor en ejecución. Se le suele agregar las opciones -it, el nombre del contenedor y el intérprete de comandos que le corresponda (/bon/sh, /sin/bash, /bash , etc). Otra cosa que se puede hacer con este comando, por ejemplo, es hacer docker exec <contenedor> <comando> para ejecutar comandos de Linux en el contenedor sin la necesidad de hacer -it

docker network ls --> lista todas las redes

docker network create <nombre> --> crea una red con el nombre que le indiquemos

docker network rm <red> --> borra una red que indiquemos

docker create --network <red> <imagen> --> crea un contenedor asignándole la red que indiquemos

docker create -e <variable>=<valor> <imagen> --> crea un contenedor asignándole un valor a la variable de entorno que indiquemos

docker tag <imagen> <nuevo-nombre> --> le cambia de nombre a una imagen que indiquemos. Lo que en realidad hace, aparentemente, es realizar una copia de la imagen con el nuevo nombre que indicamos

docker build -t <nombre>:<versión> <ruta> --> crea una imagen a partir del dockerfile presente en le ruta que indiquemos con el nombre que indiquemos y su versión. Si estamos posicionados en el mismo directorio donde está el dockerfile, el argumento de la ruta sería un punto

docker search <caracteres> --> sirve para listar las imágenes que se pueden descargar relacionada a los caracteres suministrados

docker cp <archivo> <contenedor>:<ruta> --> copia un archivo que indiquemos en el directorio del contenedor en ejecución que indiquemos

docker compose up --> construye los contendores que están en el archivo docker-compose.yml. Debemos estar posicionados en el directorio donde está dicho archivo. Con ctrl + c detenemos todos los servicios 

docker compose down --> elimina las imágenes y contenedores construidos a partir del archivo docker-compose.yml del directorio

docker compose stop --> detiene los contenedores de fueron creado y ejecutados por el archivo de docker compose 

Cuando utilizamos Docker compose, a los contenedores creados a partir del archivo docker-compose.yml se los agrupará en una red, por lo que no es necesario que incluyamos las redes en el archivo yml

docker compose -f <nombre> up --> sirve para crear los contenedores y la red a partir de un archivo  de docker compose que tenga un nombre diferente

docker compose up -d --> es para crear y ejecutar contenedores en modo "deprendido" (los contenedores se ejecutan en segundo plano)
