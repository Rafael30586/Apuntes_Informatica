Una dependencia de software es un componente de software del cual una aplicación depende para funcionar correctamente. Las dependencias de software proveen capacidades específicas como acceso a base de datos, redes, autenticación o elementos de interfaz de usuario que les permite a los desarrolladores construir aplicaciones eficientemente sin reinventar una funcionalidad común.
En simples palabras, si una pieza de software no puede ejecutarse, construirse o comportarse como se espera sin otra, tiene una dependencia.

Aspectos clave de las dependencias de software:
* Una dependencia es un biblioteca, paquete, módulo, framework o servicio usado por una aplicación.
* Las dependencias de software permite a las aplicaciones llevar a cabo funciones específicas que no pueden implementar por ellas mismas.
* Si una dependencia falta, es incompatible, o funciona mal la aplicación podría no funcionar o comportarse de manera impredecible.

-----------
Tipos de dependencias de software
Las dependencias de software entran en dos grandes clasificaciones basadas en como son introducidas en la aplicación: directas o transitivas.

Dependencias directas
Son componentes de software que los desarrolladores eligen explícitamente y declaran para su uso en un proyecto. Estas dependencias son añadidas intencionalmente para proveer de funcionalidades específicas a la aplicación.
Las dependencias directas están típicamente listadas en archivos de configuración como: package.json (node.js), pom.xml (maven), build.graddle (Gradle), etc.

Dependencias transitivas
Dependencias transitivas son componentes de software de los cuales una aplicación depende indirectamente. No son añadidas directamente por el desarrollador pero son requeridas por las dependencias directas de una aplicación para funcionar correctamente. 
En otras palabras, las dependencias transitivas son dependencias de otras dependencias.
¿Por qué son importantes?
* Están incluidas automáticamente durante el proceso de construcción.
* Generalmente componen la mayoría de las dependencias totales de una aplicación.
* Pueden introducir vulnerabilidades, riesgos de licencia o conflictos entre versiones.
* Los desarrolladores podrían no estar enterados de su presencia sin analizar el árbol completo de dependencias.
