¿Qué es AWS ECS?

Amazon Elastic Container Service es un servicio de orquestación de contenedores basado en la nube y totalmente gestionado. Te permite ejecutar tus aplicaciones. Te permite ejecutar tus aplicaciones en la nube sin tener que configurar ni mantener infraestructura.
Para asegurarte de que las demandas de capacidad se satisfacen de forma óptima y mantener el máximo rendimiento, podés configurar Amazon ECS para que evalúe y supervise continuamente los procesos de memoria y CPU. Estos datos pueden ayudarte a determinar una estrategia de despliegue óptima para cada contenedor. Además, podés aprovechar ECS para escalar contenedores y lanzar actualizaciones cuando lo necesites
ECS admite la integración con varios servicios y funciones útiles de AWS, incluyendo AWS IAM, AWS EBS y AWS ELB

-----------------------
Definiciones de tareas de AWS ECS

Para preparar tu aplicación para que se ejecute en Amazon ECS, necesitás crear una definición de tarea. Una definición de tarea es un archivo de texto (en formato JSON) que describe uno o más contenedores (hasta 10) que forman tu aplicación 
La definición del trabajo puede especificar varios parámetros para tu aplicación en contenedores. Por ejemplo, podés usar estos parámetros para indicar qué contenedores ejecutar, qué puertos deben estar abiertos, qué volúmenes de datos deben usarse, qué modo de red Docker usar y los roles de IAM
Casi todo lo que podés ejecutar en un comando de Docker de línea de comandos se puede definir usando definiciones de tareas ECS.

------------------------
AWS Tasks y programación

Una tarea es una instancia de una definición de tarea que se ejecuta en un cluster. Después de crear una definición de tarea para tu aplicación en Amazon ECS, podés especificar el número de tareas que se ejecutarán en tu cluster
El programador de tareas de Amazon ECS se encarga de colocar tareas en un cluster. Hay dos estrategias principales de programación:
* La estrategia REPLICA coloca un número deseado de tareas en tu cluster y mantiene este número de tareas, restaurándolas si algunas fallan
* LA estrategia DAEMON coloca una tarea en cada instancia de contenedor activa que cumpla los criterios relevantes

Cluster AWS ECS

Un cluster de Amazon ECS es una agrupación lógica de servicios, cada uno compuesto por una o más tareas. La primera vez que usás Amazon .ECS, se crea un cluster predeterminado. Podés crear clusters adicionales en tu cuenta para lograr el aislamiento de recursos para diferentes cargas de trabajo o proyectos
Existen dos formas de ejecutar recursos de cluster: en instancias EC2 o a través de Amazon Fargate

------------------------
Opciones de implementación de AWS ECS: EC2 vs Fargate

ECS puede utilizar instancias  de EC2 para ejecutar contenedores. Las instancias EC2 se impleentan como parte del servicio Amazon EC2 y se "registran" en un cluster ECS definido, lo que significa que EC2 puede usar esas instancias para implementar contenedores. Podés ejecutar un cluster ECS dentro de una VPC existente, haciéndolo accesible a los recursos de AWS existentes que se ejecutan en esa VPC
Una forma alternativa de desplegar contenedores ECS es con Amazon Fargate. Fargate elimina la necesidad de aprovisionar, configurar y administrar instancias EC2, permitiendo que AWS administre las instancias EC2 por vos
Con Fargate no tenés que gestionar servidores, pero debés asegurarte de que las definiciones de tareas sean sin estado. Actualmente, no podés adjuntar un volumen de almacenamiento al contenedor definido en el trabajo
Esto significa que si usás Fargate como almacenamiento persistente, tendrás que usar servicios de almacenamiento externos como S3 o RDS
Usá ECS con EC2 cuando:
* Toda tu infraestructura se ejecuta en AWS
* Ya definiste la VPC y las subredes
* Tenés procesos existentes para el aprovisionamiento y la gestión de instancias EC2
* Las aplicaciones requieren almacenamiento persistente conectado directamente
Usá ECS Fargate cuando:
* No tenés una VPC existente
* Tenés algunos recursos ejecutándose en Amazon y algunos en otros entornos de la nube
* Las aplicaciones son apátridas o no requieren volúmenes de almacenamiento conectados directamente
