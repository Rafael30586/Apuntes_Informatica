¿Qué es AWS ECS?

Amazon Elastic Container Service es un servicio de orquestación de contenedores basado en la nube y totalmente gestionado. Te permite ejecutar tus aplicaciones. Te permite ejecutar tus aplicaciones en la nube sin tener que configurar ni mantener infraestructura.
Para asegurarte de que las demandas de capacidad se satisfacen de forma óptima y mantener el máximo rendimiento, podés configurar Amazon ECS para que evalúe y supervise continuamente los procesos de memoria y CPU. Estos datos pueden ayudarte a determinar una estrategia de despliegue óptima para cada contenedor. Además, podés aprovechar ECS para escalar contenedores y lanzar actualizaciones cuando lo necesites
ECS admite la integración con varios servicios y funciones útiles de AWS, incluyendo AWS IAM, AWS EBS y AWS ELB

Definiciones de tareas de AWS ECS

Para preparar tu aplicación para que se ejecute en Amazon ECS, necesitás crear una definición de tarea. Una definición de tarea es un archivo de texto (en formato JSON) que describe uno o más contenedores (hasta 10) que forman tu aplicación 
La definición del trabajo puede especificar varios parámetros para tu aplicación en contenedores. Por ejemplo, podés usar estos parámetros para indicar qué contenedores ejecutar, qué puertos deben estar abiertos, qué volúmenes de datos deben usarse, qué modo de red Docker usar y los roles de IAM
Casi todo lo que podés ejecutar en un comando de Docker de línea de comandos se puede definir usando definiciones de tareas ECS.
