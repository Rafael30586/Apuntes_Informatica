¿Qué es AWS global Accelerator?

Es un servicio de red que mejora la disponibilidad y rendimiento de aplicaciones con usuarios globales al enrutar tráfico de usuario a través de la vasta infraestructura global de Amazon web services
Usa la red global de AWS para direccionar el tráfico al punto final más cercano y saludable basado en revisaciones de salud y ubicación geográfica. Esto ayuda a reducir latencia de internet, mejorar velocidades de transferencia e incrementar la fiabilidad de acceso de aplicaciones

Puntos clave

* AWS Accelerator soporta dos tipos de aceleradores:
  - Acelerador estándar: mejora la disponibilidad de tus aplicaciones de internet globalmente al direccionar tráfico a lo largo de la red de AWS al punto final regional más cercano
  - Acelerador de enrutamiento personalizado: Los aceleradores de enrutamiento personalizado permiten que el tráfico de usuario específico sea designado a múltiples puntos finales dentro de una subred VPC, apuntando a direcciones IP privadas

	
* El acelerador global personalizado enruta tráfico al puerto e instancia designados
* AWS global accelerator provee direcciones IP estáticas como puntos de entrada fijos para tu aplicación, reduciendo la complejidad asociada con gestionar varias direcciones IP a lo largo de diferentes regiones de AWS y zonas de disponibilidad. 
* Soporta varios tipos de puntos finales como balanceadores de carga de red, balanceadores de carga de aplicación, instancias EC2 y direcciones IP elásticas a lo largo de una o múltiples regiones de AWS	
* Podés usarlo para controlar la proporción de tráfico dirigida a cada punto final dentro de un grupo de puntos finales al asignar pesos a lo largo de los puntos finales
* Por defecto, AWS global Accelerator ofrece dos direcciones IP estáticas, o 4 direcciones (2 IPv4 y 2 IPv6) para configuraciones doble-pila. Los usuarios pueden también traer sus propias direcciones IP (BYOIP). Estas IP sirven como puntos de entrada consistentes para tus aplicaciones y permanecen con tu acelerador mientras exista, incluso si el acelerador está desactivado

¿Cómo funciona AWS Global Accelerator?

https://www.nops.io/wp-content/uploads/2024/08/end-user-request.png

1) El usuario realiza una solicitud: cuando el usuario realiza una solicitud, se dirige hacia las IPs estáticas de AWS Global Accelerator, que sirven como puntos de entrada fijos a las aplicaciones

2) Enrutamiento de tráfico: AWS Global Accelerator procesa la solicitud, aprovechando la red global de AWS. Accede a la ubicación geográfica de los puntos finales de aplicación más cercanos para determinar el camino más eficiente para enrutar tráfico

https://www.nops.io/wp-content/uploads/2024/08/aws-global-edge.png

3) Optimización de puntos finales: el servicio selecciona el punto final óptimo entre las opciones de configuración tales como balanceadores de carga de red, balanceadores de carga de aplicación o instancias EC2 a lo largo de múltiples regiones de AWS. Esta decisión se basa en factores como puntos finales de salud y proximidad geográfica para minimizar latencia

4) Procesamiento de solicitud y respuesta: El punto final elegido procesa la solicitud de usuario. La respuesta luego enrutada a través de global Accelerator al usuario, optimizando el camino de entrega para velocidad/fiabilidad 

----------------------------
Casos de uso para AWS Global Accelerator

* Escalado simple de red para aplicaciones: mientras el uso de aplicaciones se incrementa, también aumenta la demanda por direcciones IP y puntos finales. AWS Global Accelerator te ayuda a escalar tu red de manera eficiente al asociar tus recursos regionales (como balanceadores de carga e instancias EC2) a direcciones IPv4 e IPv6 estáticas

* Aplicaciones de baja latencia como VoIP, video juegos, streaming, etc

* Disponibilidad, recuperación de desastres y resiliencia: Global Accelerator asegura que tu red permanezca disponible a lo largo de múltiples regiones de AWS y zonas de red independientes, aumentando la recuperación ante desastres. Automáticamente re-enruta el tráfico a una región secundaria si la principal falla. 

* Seguridad y prevención DDoS

----------------------------
AWS Global Accelerator vs CloudFront

AWS Global Accelerator y Amazon CloudFront cumplen con roles distintos pero complementarios en gestionar tráfico de aplicaciones. Mientras Global Accelerator optimiza el enrutamiento de tráfico al punto final de AWS más cercano para minimizar latencia y mejorar rendimiento,  CloudFront es un CDN que cachea contenido más cercano a los usuarios para reducir tiempos de respuesta y costos de ancho de banda. Elegir entre ellos depedne de si tu prioridad es reducir latencia para contenido dinámico con Global Accelerator o mejorar la velocidad de entrega de contenido con CloudFront
