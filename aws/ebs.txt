AWS Elastic Block Store (EBS)

Es un servicio de almacenamiento persistente y de alto rendimiento diseñado para usarse con EC2. Funciona como un disco duro virtual para instancias EC2. Está diseñado para cargas de trabajo que requieren baja latencia, IOPS alto y almacenamiento durable
* Almacenamiento persistente  que permanece disponible incluso después de que una instancia EC2 se detiene o termina
* Entrega baja latencia y alto IOPS para cargas de trabajo transaccionales
* Replica datos automáticamente dentro de la misma zona de disponibilidad
* Soporta volúmenes elásticos para redimensión dinámica para ajuste de rendimiento

---------------------------
Características clave

* Volúmenes elásticos: Modificar tipo de almacenamiento, capacidad y rendimiento dinámicamente sin tiempo de inactividad
* Capturas (snapshots) incrementales: las capturas de EBS son respaldos incrementales almacenados internamente en Amazon S3
* Rápida recuperación de captura: quita el retraso de inicialización y entrega volúmenes completamente aprovisionados inmediatamente luego de la recuperación
* EBS multi-acople: acopla un volumen io1 o io2 a múltiples instancias EC2 en la misma zona de disponibilidad
* Alta durabilidad: provee almacenamiento altamente durable con niveles de durabilidad desde 99,8% hasta 99,999% dependiendo del tipo de volumen
* Encriptación nativa: Encripta datos en descanso, en tránsito y volúmenes acoplados usando AWS Key Management Service (KMS)

--------------------------
Tipos de volúmenes de Amazon EBS

1) Volúmenes de unidades de estado sólido (SSD). Diseñados para cargas de trabajo de bases de datos que requieren IOPS altos y rendimiento de baja latencia
* SSD de propósito general (gp3): el tipo de volumen por defecto. Ofrece latencia de 1 digito de milisegundos con una base de referencia de 3000 IOPS y un thoughput de 125 MB/s. El rendimiento puede escalar hasta 16000 IOPS y thoguhput de 1000 MB/s independientemente de la capacidad de almacenamiento
* SSD de IOPS aprovisionados (io2 Block Express): Hecho para bases de datos de alto rendimiento y críticas (Oracle, SAP, HANA). Entrega hasta 256000 IOPS, 4000 MB/s de thoughput por volumen y 99,9999% de durabilidad
* SSD de IOPS aprovisionados (io1): Volumen legacy de alto rendimiento. Entrega hasta 64000 IOPS y 1000 MB de thoughput por volumen. Diseñado para llevar a cabo dentro del 10 % de IOPS provisionados 99,9% del tiempo

2) Volúmenes de disco duro (HDD). Optimizados para cargas de trabajo de streaming de datos grandes y secuenciales donde el throughput es valorado sobre IOPS. Los volúmenes respaldados por HDD no pueden ser usados como "boot volumes"
* HDD de throughput optimizado (st1): idóneo paqra cargas de trabajo eccedidas frecuentemente y de throughput intensivo como MapReduce, Kafka, procesamiento de logs y almacenamiento de datos. Provee un burst throughput de hasta 500 MB/s
* Cold HDD (sc1): La opción de almacenamiento de menor costo, diseñado para conjuntos de datos grandes y accedidos con poca frecuencia con capacidades burst throughput de hasta 250 MB/s

--------------------------------
Capacidades avanzadas de EBS

1) EBS multi-attach
Aunque los volúmenes estándar de EBS están limitados a una instancia EC2, podés activar multi-attach en volúmenes de IOPS aprovisionados (io1 e io2). Esto permite a un volumen ser acoplado concurrentemente 16 instacias EC2 nitro-based en la misma zona de disponibilidad. Esta característica requiere sistemasd e archivos agrupados (como GSF2 o OCFS2) para coordinar acciones de escrituras seguras cooncurrentes

2) EBS en bases de datos y aplicaciones gestionadas. EBS opera como un alamcén de datos vital persistente a lo largo de servicios de bases de datos nativos de AWS y disposiciones de infraestructura:
* Bases de datos autogestionadas: sirve como almacenamiento de bases de datos altamente persistente (MySQL, PostGreSQL) acoplado a instancias EC2 individuales
* Servicios de AWS gestionados: Funciona como cimiento de almacenamiento persistente subyacente corriendo detrás de Amazon RDS y Amazon DocumentDB
* Almacenamiento de contenedores: provee montaje de volúmenes durables y persistentes para cargas de trabajo contenerizadas y con estado gestionadas por Amazon EKS y ECS
