Un arquetipo de Maven es una abstracción de un tipo de un proyecto que puede ser instanciado en un proyecto de Maven concreto y personalizado. En pocas palabras, es un proyecto plantilla desde el cual otros proyectos son creados.
El principal beneficio de usar arquetipos es estandarizar el desarrollo de proyectos y permitir a los desarrolladores fácilmente seguir las mejores prácticas y arrancando sus proyectos más rápido.

-----------------------
Maven archetype descriptor

El Maven archetype descriptor es el corazón del proyecto de arquetipo. Es un archivo XML llamado archetype-metadata.xml y localizado en el directorio META-INF/Maven.
Se usa para describir los metadatos del arquetipo:

 <archetype-descriptor
  ...
  name="custom-archetype">

    <requiredProperties>
        <requiredProperty key="foo">
            <defaultValue>bar</defaultValue>
        </requiredProperty>
    </requiredProperties>

    <fileSets>
        <fileSet filtered="true" packaged="true">
            <directory>src/main/java</directory>
            <includes>
                <include>**/*.java</include>
            </includes>
        </fileSet>
    </fileSets>

    <modules>
        <module name="sub-module"></module>
    </modules>

</archetype-descriptor>

La etiqueta requiredProperties es usada para proveer propiedades durante la generación del proyecto. Así, seremos estimulados a proveer valores para ellos, con la libertad de aceptar el valor por defecto.
fileSets, por otro lado, son usados para configurar que recursos serán copiados en el proyecto concreto generado. Un archivo "filtered" significa que marcadores de posición serán substituidos con valores proveídos durante el proceso de generación.
Y al usar packaged="true" en el fileSet, estamos diciendo que los archivos seleccionados serán añadidos a la jerarquía de carpetas especificada por la propiedad package.
Si queremos generar un proyecto multi-módulo, entonces la etiqueta modules puede ayudar a configurar todos los módulos del proyecto generado.
Nótese que este archivo es sobre archetype 2 en adelante. En la versión 1.0.x el archivo se llamaba archetype.xml y tenía una estructura diferente.

------------------------
Como crear un arquetipo

Un arquetipo es un proyecto normal de Maven con el siguiente contenido extra:
* src/main/resources/archetype-resources es la plantilla de la cual los recursos son copiados al nuevo proyecto creado
* src/main/resources/META-INF/maven/archetype-metadata.xml: es el descriptor usado patra describir los metadatos del arquetipo

Para crear un arquetipo manualmente, podemos comenzar con un nuevo proyecto creado en Maven y después podemos añadir los recursos mencionados más arriba.
O podemos generarlo al usar el archetype-maven-plugin, y luego personalizar el contenido del directorio archetype-resources y el archivo archetype-metada.xml
Para generar el arquetipo podemos usar:
 
mvn archetype:generate -B -DarchetypeArtifactId=maven-archetype-archetype \
  -DarchetypeGroupId=maven-archetype \
  -DgroupId=<groupId> \
  -DartifactId=<artifactId>

También podemos crear el arquetipo de un proyecto en Maven existente:
  mvn archetype:create-from-project

Se genera en target/generated-sources/archetype listo para ser usado

No importa como hayamos creado el arquetipo, terminaremos con la siguiente estructura

 archetype-root/
├── pom.xml
└── src
    └── main
        ├── java
        └── resources
            ├── archetype-resources
            │   ├── pom.xml
            │   └── src
            └── META-INF
                └── maven
                    └── archetype-metadata.xml

Ahora podemos comenzar a construir nuestro arquetipo al colocar recursos en el directorio archetype-resources y configurarlos en el archivo archetype-metadata.xml

---------------------------------
Construyendo el arquetipo

Ahora estamos listos para personalizar nuestro arquetipo. Ahora ejemplificaremos este proceso con la creación de un simple arquetipo de Maven para generar aplicaciones RESTful basado en JAX-RS 2.1. Llamémoslo maven-archetype

* Empaquetamiento de arquetipo

Comenzamos por modificar el pom.xml del proyecto de arquetipo localizado bajo el directorio maven-archetype

<packaging>maven-archetype</packaging>

Este tipo de empaquetamiento está disponible gracias a la extensión archetype-packaging 

<build>
    <extensions>
        <extension>
            <groupId>org.apache.maven.archetype</groupId>
            <artifactId>archetype-packaging</artifactId>
            <version>3.0.1</version>
        </extension>
    </extensions>
    <!--....-->
</build>

* Añadiendo el pom.xml

Ahora creamos un archivo pom.xml localizado bajo el directorio archetype-resources:

<project ...>

    <groupId>${groupId}</groupId>
    <artifactId>${artifactId}</artifactId>
    <version>${version}</version>
    <packaging>war</packaging>

    <dependencies>
        <dependency>
            <groupId>jakarta.ws.rs</groupId>
            <artifactId>jakarta.ws.rs-api</artifactId>
            <version>2.1.6</version>
            <scope>provided</scope>
        </dependency>        
    </dependencies>

</project>

Como podemos ver, groupId, artifactId y version están parametrizados. Serán reemplazados durante la creación de un nuevo proyecto de este arquetipo.
Podemos colocar todo lo necesario por el proyecto generado, como dependencias y plugins en el pom.xml. Acá, hemos añadido la dependencia JAX RS como el arquetipo que será usado para generar aplicaciones basadas en RESTful

* Añadiendo los recursos requeridos

Luego, podemos añadir código java para nuestra aplicación en archetype-resources/src/main/java
Una clase para configurar la aplicación JAX-RS:

package ${package};
// import
@ApplicationPath("${app-path}")
public class AppConfig extends Application {
}

Y una clase para el recurso ping: 

@Path("ping")
public class PingResource{
    //...
}

Finalmente, poner el archivo de configuración del servidor, server.xml, en archetype-respurces/src/main/config/Liberty

* Configurando metadatos

Después de añadir todos los recursos necesarios, podemos ahora configurar cuales serán copiados durante la generación a través del archivo archetype-metadata.xml

<archetype-descriptor
  name="maven-archetype">
    
    <!--...-->
    <fileSets>
        <fileSet filtered="true" packaged="true">
            <directory>src/main/java</directory>
            <includes>
                <include>**/*.java</include>
            </includes>
        </fileSet>
        <fileSet>
            <directory>src/main/config/liberty</directory>
            <includes>
                <include>server.xml</include>
            </includes>
        </fileSet>
    </fileSets>

</archetype-descriptor>

Acá queremos que todos los archivos java del directorio src/main/java y el archivo server.xml de src/main/config/Liberty se copien
