¿Qué es java Enterprise edition?

Java Enmterprise edition (Java EE) también conocido como Jakarta EE, se usa para diseñar aplicaciones grandes y poderosas usadas por organizaciones. Provee herramientas y funciones que ayudan a los desarrolladores crear sitios web, aplicaciones y sistemas backend que pueden encargarse de muchos usuarios al mismo tiempo. JEE incluye soporte integrado para desarrollo web, gestión de bases de datos y seguridad (security).
JEE is ampliamente utilizado en industrias como la banca, salud y e-commerce por su fiabilidad, escalabilidad y seguridad. JEE continúa haciendo que el desarrollo de software empresarial sea más eficiente.

---------------------------
Arquitectura de java Enterprise edition

Java EE tiene una arquitectura multicapa, lo que quiere decir que la aplicación está dividida en diferentes capas. Esto hace que sea más fácil gestionar, escalar y actualizar sin afectar al sistema entero

https://media.geeksforgeeks.org/wp-content/uploads/20250211173623531043/Architecture.webp

1. Capa de cliente
* Esta es la capa del frontend adonde los usuarios interactúan con la aplicación
* Puede ser un buscador web, aplicación móvil o de escritorio
* Usa tecnologías como HTML, CSS, JavaScript y JSP (Java server pages)

2. Capa web (capa servlet)
* La capa web se encarga de las solicitudes de usuario y respuestas
* Usa servlets, JSP, JSF para procesar datos de los usuarios
* Envía los datos a la capa de negocio para su procesamiento

3. Capa de negocio (capa de lógica)
* La capa de negocio contiene la lógica nuclear de una aplicación
* Usa EJB (Enterprise java beans) para manejar tareas como cálculos, transacciones y seguridad.
* Procesa datos y se comunica con la capa de bases de datos

4. Capa de datos
* La capa de datos almacena y extrae datos de la base de datos
* Usa JPA y JOBC para gestión de bases de datos
* Asegura almacenamiento de datos seguro y eficiente

5. Capa de integración (opcional)
* Se usa para conectar la aplicación con sistemas externos, APIs y servicios
* Usa java messaging services (JMS) y servicios web

Componentes nucleares de JEE

1. Servlets: es una parte del marco de trabajo de JEE. Se usan para desarrollo web. Los servlets se encargan de procesar las solicitudes de clientes y generar respuestas. Se usan para extender funciones de servidores web al recibir y responder a solicitudes de clientes web

2. JSP: también conocido como java server pages, permite incorporar código java dentro del HTML para páginas web dinámicas. Es más fácil de mantener las JSP porque ayudan a distinguir entre diseño y desarrollo. JSP provee algunas funciones añadidas como lenguaje de expresión, etiquetas personalizadas, etc

3. JSF: JSF es por java server faces. Es un marco de trabajo usado para construcción de interfaces de usuario basadas en web. Hace que el desarrollo web sea más simple al proveer componentes de UI como formularios, botones y tablas. Se usa para gestionar beans y manejar datos y acción efiientemente

4. EJB: EJB es por Enterprise java beans. Maneja lógica de negocio compleja, transacciones y seguridad. EJB ayuda a los desarrolladores a crear lógica de negocio segura, escalable y reutilizable para grandes aplicaciones. Se usa principalmente en bancos, e-commerce, etc.

5. CDI: CDI es por contexts and dependency injection que se usa para simplificar gestión de dependencias y se encarga del ciclo de vida de objetos. Simplifica la dependencia entre diferentes partes de una aplicación. Hace a las aplicaciones java más flexibles, modulares fáciles d emantener

6. JAAS: JAAS se conoce como java authentication and authorization service, es un marco de trabajo de seguridad de java que se usa para autenticar usuarios y acceso de control en aplicaciones. Ayuda a asegurar que solo usuarios autorizados puedan acceder a ciertas partes del sistema. Verifica la identidad de los usuarios usando un nombre de usuario, contraseña y otros métodos

7. JDBC: JDBC es por java database connectivity, una API de java que permite que las aplicaciones se conecten a bases de datos, envíen consultas y extraigan datos. Ayuda a los programas java a interactuar con bases de datos como MySQL, Oracle, SQL Server. Solicita y procesa datos de una base de datos

8. Beans gestionados (managed beans): es una simple clase de java que es automáticamente creada y gestionada por el contenedor java EE. Se usa principalmente en JSF y CDI para manejar lógica de negocio e interacciones de usuario en aplicaciones web. Retiene input de usuario y procesa acciones

9. Validación de beans: se usa para validar input de usuario y datos en aplicaciones java. Asegura que los datos sigan reglas específicas. Revisa si los valores de input cumplen con las condiciones requeridas. La validación de beans usa anotaciones simples como @NotNull, @Size y @Pattern

10 Application client: Un application client en java EE es un programa java autónomo que corre en una computadora de usuario y se conecta al servidor java EE para usar servicios empresariales como EJB, JPA, JMS. Corre separado de las aplicaciones web. Puede extraer o enviar datos a las aplicaciones principales
