¿Qué es un contenedor web, servidor de aplicación y servidor web?

Servlet: es cualquier clase de java que pueda aceptar una solicitud, procesarla y devolver una respuesta. Los servlets son clases de java ejecutadas en el servidor y se usan para generar páginas web dinámicas. Todos los servlets se registran a sí mismos con contenedores servlet

Servlet/Contenedor web: Una parte del servidor web es responsable de gestionar todos los servlets en el sistema. Ejemplo: Tomcat, Grizzly o Jetty. En el inicio del sistema, el contenedor de servlet crea un objeto de ServletContext. El rol de este objeto es recordar todos los servlets, filtros y escuchadores asociados con la aplicación web. El ServletContext vive mientras el contenedor esté activo
En cada nueva solicitud del cliente, el contenedor de servlets crea nuevos objetos HttpServletRequest y HttpServletResponse y los pasa al método service() del servlet

Servidor de aplicación: Un contenedor web como Apache Tomcat o jetty implementa la especificación de JSP y servlet de java EE. No implementa especificaciones EJB. Por eso no podemos desplegar y ejecutar aplicaciones EJB en un contendor de servlet. En su lugar, necesitamos un servidor de aplicaciones completo. Un servidor de aplicaciones contiene un contenedor EJB así como también un contenedor de servlet e implementa todas las especificaciones requeridas de java EE. Por ejemplo: Glassfish application server.
Con frameworks de java livianos como spring y hibernate, no necesitamos un servidor de aplicaciones completo. Un contenedor web como Tomcat puede ser usado en su lugar con solo agregar los jars de bibliotecas requeridos al classpath

Servidor Web/HTTP: un servidor web es software que ayuda a entregar contenido web (páginas web) a los clientes (por ejemplo, navegadores web) a través de internet usando el protocolo HTTP. Un servidor HTTP puede encargarse solo de archivos HTML y no puede ejecutar servlets o JSP. Ejemplos: Apache HTTP server, NGINX

Diferencia entre contenedor servlet/web, servidor de aplicación y servidor web/HTTP: un servidor HTTP puede manejar solamente archivos HTML. Un contenedor web o contenedor de servlet puede manejar servlets y JSPs. Un servidor de aplicación contiene un contenedor web y un contenedor EJB y puede manejar servlets, JSPs y EJBs. Los EJBs solo pueden ser desplegados en servidores de aplicación, pero no en contenedores web. Spring y hibernate son alternativas livianas a EJBs y pueden ser desplegados en un contenedor web
