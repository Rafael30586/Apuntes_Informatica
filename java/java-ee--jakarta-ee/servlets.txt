Un servlet java es un programa java que corre en un servidor web habilitado para java o en un servidor de aplicación. Se encarga de las solicitudes del cliente, las procesa y genera respuestas dinámicamente. Los servlets son la columna vertebral de muchas aplicaciones java del lado del servidor debido a su eficiencia y escalabilidad.

------------------
Características de los servlets de java
* Funcionan del lado del servidor
* Se encargan eficientemente complejas solicitudes del cliente
* Generan respuestas dinámicas
* Ofrecen mejor rendimiento comparado a tecnologías más viejas como CGI
* Altamente escalables en aplicaciones web de nivel empresarial

------------------
Arquitectura de los java servlets
Los contenedores de los java servlet juegan un papel muy importante. Es responsable de encargarse de tareas importantes como balanceo de carga, gestión de sesión, asignación de recursos; se asegura de que todas las solicitudes se procecen eficientemente bajo elevado tráfico

https://media.geeksforgeeks.org/wp-content/uploads/20250212150508046406/Jsp-servlet-architecture.webp

Flujo de trabajo de la arquitectura de servlet
* Los clientes envían la solicitud al servidor web
* El servidor web recibe la solicitud
* El servidor web pasa la solicitud al servlet correspondiente
* El servlet procesa la solicitud y genera la respuesta en la forma de output
* El servlet envía la respuesta de vuelta al servidor web
* El servidor web envía la respuesta de vuelta al cliente y el browser del cliente la muestra en pantalla	

-------------------
Contenedor de servlet (servlet container)
También conocido como motor de servlet (servlet engine), es un conjunto de objetos integrado que ofrece un entorno de ejecución para componentes de java servlet. Es un sistema que gestiona componentes de servlets java sobre el servidor web para encargarse de las solicitudes del cliente web.

Servicios proveídos por el servlet container:
* Servicios de red: carga la clase servlet. La carga podrá ser de un sistema de archivos local, un sistema de archivos remoto u otros servicios de red. El servlet container provee los servicios de red sobre los que las solicitudes y respuestas son enviadas
* Provee el servicio de codificar y decodificar mensajes MIME-based
* Gestiona el ciclo de vida de un servlet
* Gestiona los recursos estáticos y dinámicos, como archivos HTML, servlets y páginas JSP
* Se encarga de autorización y autenticación de acceso a recursos
* Mantiene una sesión al adjuntar un ID de sesión a la ruta de la URL

------------------
Pre-requisitos para crear un servlet java
* Instalar el JDK
* Instalar el servidor Apache Tomcat (por ejemplo, versión 9 o 10)
* Instalar un IDE como IntelliJ IDEA o Eclipse
* Descargar el JAR API servlet (ya incluido en Tomcat)

----------------
¿Por qué elegir servlets java en lugar de otras tecnologías?
El contenido web dinámico requiere tecnologías del lado del servidor. Aunque hay muchas opciones, los servlets java se destacan debido a sus ventajas sobre alternativas como CGI(Common Gateway Interface)

Limitaciones de CGI
* CGI crea y destruye un proceso por casa solicitud del cliente, llevando a un alto consumo de recursos.
* Problemas de escalabilidad: Rendimiento pobre cuando hay muchas solicitudes del cliente
