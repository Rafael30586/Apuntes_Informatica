JSP es más simple que usar servlets, pero esto no quiere decir que los reemplacen completamente. Tomcat es un contenedor de servlets, estos son necesarios para el desarrollo de una aplicación web con java EE o jakarta EE. Los JSPs son convertidos en servlets durante la ejecución de la aplicación

Cuando trabajamos con el modelo vista controlador usando jakarta EE, el controlador es un servlet, las vistas son archivos JSP y los modelos son POJOs. Los controladores tienen la lógica de negocio (no hay que colocarla en los JSP). Los controladores se conectan a un servlet que sería el service, el service se conecta a una clase DAO y la clase DAO a una base de datos. Tener en cuenta que el servicio y DAO no forman parte del modelo vista controlador
