¿Qué es una sesión web?

Una sesión web es la cantidad de tiempo que un usuario utiliza navegando un sitio web; desde el momento en que llegan al sitio web, hasta el momento que dejan el sitio

------------------------
¿Cómo funcionan las sesiones web?

En la práctica, una sesión web consiste en los datos o archivos que persisten a través del uso de un sitio web o aplicación web. Estos recursos son identificados con un id de sesión. Al navegador del usuario se le es dado este id al empezar una nueva sesión, y este id es intercambiado durante cada comunicación subsecuente entre el navegador y el servidor
Una sesión web incluye toda la información que podría ser relevante durante la visita del usuario. Dependiendo del propósito del sitio web o aplicación web, esto podría incluir datos como:
* Las páginas que el usuario ha visto
* Los detalles de contacto que el usuario ha ingresado en un formulario
* Los ítems en el carrito de compras del usuario

Típicamente hay un tiempo límite para las sesiones web, después de la cual se le acabará el tiempo a la sesión. Este tiempo máximo depende de la implementación. Por ejemplo, sesiones web en Google analytics expiran luego de 30 minutos de inactividad; la posterior actividad de usuario será tratada como una nueva sesión
Los sitios web y aplicaciones web con muchos usuarios frecuentemente cachean sesiones web, para que puedan ser recuperadas más rápido y de manera más eficiente

---------------------------
¿Por qué usar sesiones web?

Las sesiones web permiten al sitio web tener memoria a corto plazo sobre la actividad del usuario. Las sesiones de corto plazo le dan a los sitios web y aplicaciones web la suficiente "memoria" para construir experiencias atrapantes y personalizadas durante el curso de una visita de usuario, sin almacenar demasiada información que es irrelevante en el largo plazo

----------------------
Sesiones web vs cookies

Aunque las cookies y las sesiones almacenan información sobre el usuario, sus funciones son diferentes en la práctica
Las cookies son archivos de texto que se usan para autenticar y rastrear visitantes a un sitio web, y que solo se almacenan en la máquina del usuario. El tiempo de vida de una cookie es típicamente mucho más largo que el de una sesión web, en el orden de meses o incluso años. Las cookies son la manera en que los sitios web almacenan información de usuario de largo plazo que debería ser preservada: por ejemplo, automáticamente logueando al usuario cuando llega al sitio web o llenando automáticamente un formulario con detalles de usuario
En contraste, las sesiones web están dirigidas a almacenar información sobre las actividades más recientes del usuario. Las sesiones web son almacenadas en el servidor, lo que previene que usuarios maliciosos las editen. Tanto las sesiones web como las cookies pueden ser usadas combinadas para rastrear el comportamiento a corto plazo y a largo plazo del usuario
