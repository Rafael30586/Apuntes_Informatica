CIDR (Classless inter domain routing) es un método de asignación de dirección IP y enrutamiento que permite un uso más eficiente de direcciones IP. A diferencia del direccionamiento basado en clase, CIDR asigna direcciones IP basándose en un prefijo de red más que en una clase (A,B o C)

Notación CIDR: a.b.c.d/n
* n = número de bits en el prefijo de la red
* Ejemplo: 192.168.1.0/24 , los primeros 24 bits son la red, y los 8 restantes son el id del anfitrión

----------------------
¿Por qué CIDR?

El "classfull addressing" desperdicia direcciones IP:
* Clase A: 2^24 IPs disponibles ; (2^24 - 2) anfitriones ; demasiado grande para pequeñas organizaciones
* Clase B: 2^16 IPs disponibles ; (2^16 - 2) anfitriones ; desperdicia 49150 anfitriones cuando se necesitan 214
* Clase C: 2^8 IPs disponibles ; (2^8 - 2) anfitriones ; solo para redes pequeñas

Problema: las organizaciones generalmente necesitan un número de anfitriones que no coincide con el tamaño de la clase, llevando a un desperdicio
Solución: CIDR permite una asignación de bloque flexible coincidiendo con los requerimientos exactos

---------------------
Reglas para formar bloques CIDR
* Todas las IPs deben ser contiguas
* El tamaño de un bloque debe ser una potencia de 2 (2^n). Simplifica la división de red
* La primera IP del bloque, debe ser divisible por el tamaño del bloque. Los últimos n bits de la primera IP del bloque deben ser 0, siendo n la potencia a la que está elevada 2 en el punto anterior. Los n bits restantes se llaman resto (remainder) según un video de redes de freecodecamp, y los primeros son el cociente (quotient). Además, la primera IP del bloque es el ID del bloque. El video es: https://www.youtube.com/watch?v=fQbBPa0ADvs&list=PLH4-9MlhMdT1uxDwPGEi3zPI0CdI2UnH2&index=16

Ejemplo : si el tamaño del bloque es 2^5 entonces el id del anfitrión contiene 5 bits y la red contiene 32 - 5 = 27 bits

https://media.geeksforgeeks.org/wp-content/uploads/20190313121313/b31.jpg

La primera dirección IP del bloque debe ser divisible por el tamaño del bloque. En simples palabras, la parte menos significante debería siempre comenzar con ceros en el id del anfitrión. Como todos los bits menos significativos del id del anfitrión es cero, entonces podemos usarlo como parte de bloque de id

Ejemplo: revisar si 100.1.2.32 a 100.1.2.47 es un bloque de direcciones IP válido o no
* Todas las direcciones IP son contiguas
* El número total de direcciones IP en el bloque es 16 = 2^4
* La primera dirección IP es 100.1.2.00100000 . El id del anfitrión contiene los últimos 4 bits y todos los 4 bits menos significativos son cero. Por consiguiente, la primera dirección IP es divisible por el tamaño del bloque. 

Las tres reglas se cumplen por este bloque. Por lo tanto, es un bloque de direcciones IP válido.

-----------------------
Ventajas de CIDR
* Uso eficiente de direcciones IP: minimiza el desperdicio de direcciones IPv4. Cabe aclarar que el desperdicio de direcciones IP es posible si la cantidad de direcciones que se necesitan no es una potencia de 2, por ejemplo, si necesitamos 5 direcciones IP, necesitaremos un bloque de 8 , que es 2^3
* Asignación flexible: soporta redes de cualquier tamaño
* Enrutamiento mejorado: junta direcciones para un enrutamiento más simple y rápido
* Baja la sobrecarga administrativa: gestión de IP y red más fácil 

Desventajas de CIDR
* Complejidad: CIDR es más complejo de implementar y gestionar comparado con el "addressing" tradicional basado en clase
* Problemas de compatibilidad: algunos dispositivos de red más viejos podrían no soportar CIDR
* Inquietudes de seguridad: implementar medidas de seguridad como reglas de firewall y listas de control de acceso puede ser más difícil
