Definición y conceptos clave

Una máscara de red es un valor de 32 bits para IPv4 o de 128 bits para IPv6 que divide una dirección IP en la porción de red y la del anfitrión. La máscara consiste en 1s binarios contiguos seguidos por 0s binarios contiguos. Los 1s identifican la porción de red, mientras que los 0s identifican la porción del anfitrión.
Las máscaras de red permiten a los dispositivos determinar si una dirección IP destino reside en la misma área local o requiere enrutamiento a una red remota. Esta operación fundamental conduce todas las decisiones de enrutamiento de IP.

Componentes esenciales
* Dirección IP: un identificador numérico único asignado a cada dispositivo en una red. Toda dirección IP requiere una máscara de red asociada para funcionar apropiadamente.
* Dirección de red (ID de red): la porción de una dirección IP de la red específica o subred. Este valor se mantiene constante para todos los dispositivos dentro del mismo segmento de red.
* Dirección del anfitrión (ID del anfitrión): la porción que identifica un dispositivo específico dentro de una red. Cada dispositivo deberá tener una dirección de anfitrión única dentro de su segmento de red.
* Representación binaria: las máscaras de red son mejor entendidas en forma binaria. La estructura binaria revela la red exacta y los límites del anfitrión.
* Bitwise AND operation: La operación lógica llevada a cabo entre una dirección IP y máscara de red para derivar la dirección de red. Esta operación crea la base de todos los cálculos de enrutamiento.
* Dotted-decimal notation: La representación común para máscaras de red IPv4, como 255.255.255.0. Este formato convierte la máscara binaria en cuatro octetos decimales.
* Notación CIDR: Classless Inter-Domain Routing (CIDR) notation provee una representación usando una barra y número (por ejemplo /24). El número indica cuantos bits representan la porción de red.

¿Cómo funcionan las máscaras de red?

Las máscaras de red operan a través de operaciones matemáticas precisas que determinan los límites de la red y decisiones de enrutamiento

Definiendo límites de red
La máscara determina el tamaño de la red y las direcciones disponibles de anfitriones. Una máscara de red /24 (255.255.255.0) provee un total de 256 direcciones con 254 direcciones de anfitriones utilizables. Las direcciones de red y broadcast están reservadas. 
Máscaras más largas crean redes más pequeñas con menos direcciones de anfitriones. Una máscara /30 provee solo 4 direcciones con dos direcciones de anfitriones utilizables, comúnmente usadas para enlaces point-to-point.

Determinación local vs remota
Los dispositivos usan un proceso sistemático para determinar el envío del paquete:
1. El dispositivo lleva a cabo un bitwise AND operation entre su dirección IP y la máscara de red
2. La misma AND operation es llevada a cabo en la dirección IP destino
3. Si la dirección de red resultante coincide, el destino es local
4. Si son diferentes, el destino es remoto y requiere enrutamiento

Métodos de comunicación
* Comunicación directa: cuando el destino es local, el dispositivo intenta comunicación directa usando  ARP para obtener la dirección MAC del destino
* Enrutamiento: Cuando el destino es remoto, el dispositivo envía el paquete a la puerta de entrada por defecto. Luego, el router toma las decisiones de envío basadas en su tabla de enrutamiento

Implementación de subnetting
Las máscaras de red facilitan el subnetting al dividir redes más grandes en pequeños segmentos más manejables. El subnetting mejora el rendimiento de la red, la seguridad y el control administrativo.
Las organizaciones pueden crear múltiples subredes desde una sola asignación de red, cada una con su propio broadcast domain y políticas de seguridad

Decisiones de enrutamiento
Los routers usan máscaras de red en tablas de enrutamiento para determinar la interfaz saliente	para paquetes IP. El prefix match más largo determina la mejor ruta para envío de paquetes
