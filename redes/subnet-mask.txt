Una máscara de subred es un número que distingue la dirección de la red y la dirección del anfitrión en una dirección IP. Una subred es una red más pequeña dentro de una red que requiere una máscara de subred.
El subnetting es el proceso de dividir una red en dos o más subredes. Su función principal es la de hacer que el enrutamiento sea más eficiente y seguro. El subnetting también ayuda a hacer un mejor uso de direcciones IPv4.
Cuando conectás un dispositivo a una red, la red le asigna una dirección IP al dispositivo. La dirección IP consiste en dos partes: la porción de red y la porción del anfitrión (host). La porción de red de la dirección IP identifica a la red mientras que la porción del anfitrión identifica al dispositivo.
La dirección de red y la dirección del anfitrión contenidas en una IP son indistinguibles la una de la otra sin una máscara de sub red (subnet mask). La máscara de subred permite al tráfico de la red entender la dirección IP al partirlas en la dirección de red y la del anfitrión.

----------------
Diferencia entre subred y máscara de subred
* Subred: describe la parte específica de una red que comparte la IP de red pero que tiene una única dirección de subred
* Máscara de subred: diferencia entre la porción de una dirección IP que se refiere a la subred y la porción que se refiere al anfitrión  
---------------
¿Por qué usar subredes?

Implementar subredes y máscaras de subred beneficia a las organizaciones para mejorar la seguridad y optimizar el desempeño para cumplir con sus necesidades de negocio efectivamente

* Enrutamiento más eficiente de datos
El tráfico broadcast crea una gran ineficiencia en redes. Este tipo de tráfico involucra que los paquetes de datos atraviesen la red por cada nodo. Al dividir los dominios de broadcast, las subredes reducen el número de nodos con el cual el tráfico de broadcast interactúa y hace que el enrutamiento de paquetes de datos sea más directo y eficiente.
Por ejemplo, imaginá un ISP proveyendo de servicios a clientes en una zona geográfica muy amplia. Al hacer subnetting, el ISP puede crear redes más pequeñas para usuarios en diferentes regiones.
En este ejemplo las subredes son la clave,  ya que la máscara de subred es lo que asegura que el tráfico esté contenido dentro de su subred designada. Esto reduce la congestión en la red.
El subnetting reduce efectivamente la distancia que los paquetes de datos tienen que recorrer en la red. Esto también ayuda a la red a enrutar datos de manera más eficiente. Todo esto incrementa la velocidad y el desempeño de la red.

* Mejor seguridad (security) de red
Si estás operando una red local grande, con muchos dispositivos conectados y un alto volumen de tráfico,  hay un número de ventajas en seguridad (security) de correr múltiples subredes en lugar de una grande.
Si una red es comprometida por un ataque, solo ese segmento se verá afectado. Usando subredes, en lugar de que todos los dispositivos en la misma red sean visibles para los atacantes, solo tendrían acceso a otros dispositivos de la misma subred.
Un jefe de redes que divide un red de área local en redes más pequeñas puede también controlar el flujo de tráfico a través de mapas de ruta, QoS o ACLs, lo cual ayuda a identificar amenazas más eficientemente.

* Aumenta la vida útil de IPv4
En lugar de asignar una única dirección IP a cada dispositivo, el subnetting permite que múltiples direcciones IP privadas existan en una única red IP. Esto significa que cada dispositivo en una red IP tiene una dirección IP con una parte que refiere a la dirección IP de la red entera, y una parte de ella refiere a la subred.

-----------------------
¿Cómo funcionan las subredes?
Las redes usan direcciones de protocolos de internet para identificar piezas de hardware específicas, como servidores o computadoras. El subnetting permite que una red con una sola dirección IP pública sea partida en redes más pequeñas. Cada una con una dirección IP interna única.
Las redes son capaces de conectar cientos e incluso miles de dispositivos. Como resultado, el tráfico podría viajar por una ruta compleja a la dirección IP asociada para un dispositivo específico.
Subnetting crea sub-redes dentro de una sola red física, limitando el uso de direcciones IP a unos pocos dispositivos. Debido a esto, los datos pueden viajar más directamente a su dirección IP destino. No necesita interactuar con todas las partes de la red.
Para identificar el dispositivo al que un paquete de datos está yendo, es importante ordenar la dirección IP de una manera lógica. El propósito es asegurar que sea fácil identificar la dirección de red y la del anfitrión.
Para hacer que el subnetting sea posible, y para estar seguros de que el tráfico llegue al destino correcto, cada dirección IP necesita ser emparejada a una máscara de subred.

------------------------
¿Cómo se relacionan las clases de IP y las máscaras de sub red?
Las redes vienen en diferentes tamaños. Algunas requieren acceso para solo algunos anfitriones. Otras necesitan ofrecer acceso a miles de anfitriones.
Por esto hay diferentes clases de direcciones IP. Existen las redes de clase A, las de clase B y las de clase C. También existen las de clase D y E. Las D facilitan el multicasting y las E la investigación.
De todas maneras, hoy en día la mayoría de las direcciones IP no tienen clase.
La clase de dirección IP determina el posible tamaño de la red, así como cuantos octetos están dedicados a la máscara de subred. Las diferentes clases de direcciones IP son adecuadas para diferentes necesidades.
* Clase A: la red soporta más de 65536 anfitriones. La máscara de subred refleja la dirección de red en el primer octeto, con el segundo, el tercero y el cuarto libres para que el administrador asigne a anfitriones y subredes.
* Clase B: la red tiene una dirección IP donde la máscara de subred refleja la dirección de la red en el primer y segundo octeto. Dejando solo al tercero y al cuarto para asignar en la red. La clase B tiene disponibilidad para conexiones de entre 256 y 65534 anfitriones.
* Clase C: esta red soporta menos anfitriones. El primero, segundo y tercer octeto son usados para reflejar el número de red. Dejando al último octeto para permitir hasta 254 direcciones de anfitriones.

Tanto las direcciones de clase B como las de clase C se ven iguales. Por lo tanto, requieren de una máscara de subred para identificar la dirección de red y la dirección del anfitrión.
