UNIX es un sistema operativo multiusuario y multitarea diseñado para proveer un entorno computacional estable, seguro y eficiente. Fue desarrollado originalmente por laboratorios AT&T Bell y luego se convirtió en la base para muchos sistemas operativos.
* Soporta la ejecución de múltiples tareas o procesos simultáneamente
* Es bien conocido por su fiabilidad, diseño estructurado y estabilidad a largo plazo
* Sus principios de diseño han influido muchos sistemas operativos como Linux, BSD y macOS

--------------------
Características clave

1. Sistema multiusuario
* Múltiples usuarios pueden acceder al sistema al mismo tiempo sin afectar los procesos de otro
* Cada usuario obtiene permisos y entorno separado para asegurar la privacidad y operaciones seguras

2. Capacidad multitarea
* UNIX puede ejecutar varios procesos simultáneamente a través gestión (scheduling) de procesos eficiente
* Tareas en primer y segundo plano pueden correr juntas, mejorando productividad y utilización de recursos

3. Portabilidad
* Escrito principalmente en lenguaje C, permitiendo el movimiento fácil de UNIX a diferentes plataformas de hardware
* Solo unos mínimos cambios son requeridos para adaptar UNIX a sistemas nuevos, haciéndolo ampliamente soportado 

4. Sistema de archivos jerárquico
* Los archivos y directorios están ordenados en un estructura como de árbol, comenzando por la raíz /
* La estructura hace a la navegación, organización de archivos y acceso a datos simple y sistemática

5. Permisos y seguridad (security)
* UNIX asigna permisos (lectura, escritura y ejecución) a usuarios, grupos y otros para protección de archivos
* La autenticación a través nombres de usuario y contraseñas asegura control de acceso seguro

6. Soporte para Shell y scripting
* La Shell actúa como un intérprete de comandos, permitiendo a los usuarios comunicarse con el sistema
* El Shell scripting automatiza tareas repetitivas, haciendo que la gestión del sistema sea más rápida y fácil

-----------------------------
Estructura en capas de UNIX

La estructura en capas de UNIX organiza sus componentes de manera jerárquica para asegurar un funcionamiento del sistema modular, seguro (secure) y eficiente

https://media.geeksforgeeks.org/wp-content/uploads/20260122104403667600/hardware.webp

* Capa 1: Hardware
  - Representa los componentes físicos de la computadora, como el CPU, memoria, dispositivos de almacenamiento y hardware de entrada/salida
  - Todas las operaciones de sistema dependen de esta capa

* Capa 2: kernel
  - El kernel es el núcleo de UNIX e interactúa directamente con el hardware
  - Gestiona tareas esenciales como asignación de memoria, programación de procesos, manejo de archivos, control de dispositivos y gestión general de recursos

* Capa 3: Programas de sistema, comandos y utilidades
  - Esta capa incluye compiladores, editores, procesadores de texto y Shell
  - También contiene herramientas comunes de UNIX como wc, grep, date, who y a.out, que cuentan con el kernel para llevar a cabo diferentes funciones como procesamiento de texto, gestión de archivos y monitoreo del sistema

* Capa 4: Capa de aplicación
  - Representa aplicaciones externas desarrolladas para el usuario que corren por encima de las utilidades del sistema y entorno de Shell 
  - Estas aplicaciones usan capas subyacentes (kernel y utilidades de sistema) para ejecutar tareas exitosamente y proveer servicios al usuario

Historia de UNIX

La historia de UNIX traza su evolución desde un simple proyecto de investigación en Bell labs a uno de los sistemas operativos más influyentes en el mundo

1. Desarrollo inicial
* Comenzó entre 1969 y 1971 en AT&T Bell labs por Ken Thompson, Dennis Ritchie y su equipo
* La primera versión de UNIX fue desarrollada en una máquina PDP-7
* Fue escrito completamente en lenguaje ensamblador, haciéndolo muy dependiente del hardware

2. Reescrito en C
* En 1973, UNIX fue completamente reescrito en el lenguaje de programación C
* Esto hizo que el sistema operativo fuera portable a través de diferentes plataformas de hardware
* Su flexibilidad ayudó a UNIX ganar popularidad en entornos académicos y de investigación

3. Expansión y adopción
* Durante los mediados de los 70 hasta 1980, universidades como UC Berkeley crearon sus propias versiones de UNIX
* Esto llevó al desarrollo de BSD UNIX, añadiendo características y herramientas avanzadas
* AT&T introdujo UNIX System V, resultando en dos familias principales de UNIX: BSD Y System V

4. Comercialización
* Desde los 1980s hasta los 1990s, muchas compañías tecnológicas construyeron sus variantes comerciales de UNIX
* Versiones populares incluyeron Solaris (Sun), HP-UX (HP), AIX (IBM) y DEC UNIX
* UNIX se volvió ampliamente popular en servidores empresariales, laboratorios de investigación y workstations

5. Influencia en sistemas operativos modernos
* En los 1990s, UNIX inspiró la creación de Linux (1991) por Linus Torvalds
* Sistemas modernos como macOS, Android y variantes de BSD siguen los principios de diseño de UNIX
* Su arquitectura continua dando forma a sistemas operativos comerciales de código abierto de hoy en día

5. Estandarización y legado
* Estándares como POSIX fueron creados para asegurar compatibilidad a través de sistemas UNIX
* Los conceptos de UNIX permanecen profundamente arraigados en redes, servidores y computación empresarial
* Su diseño continúa influenciando a la computación moderna, asegurando un legado fuerte y de larga duración

El kernel en UNIX

El kernel es el núcleo y la parte más esencial de UNIX. Funciona como la unidad central de control que gestiona todos los recurso del sistema y permite la comunicación entre hardware y software.
* El kernel interactúa directamente con el hardware y lleva a cabo tareas de bajo nivel como asignación (allocation) de memoria, control de dispositivos y programación de CPU
* Gestiona procesos, archivos y system calls, asegurando que el sistema corra de manera fluida y segura
* Pr9ovee una interfaz para aplicaciones para solicitar servicios de sistema sin acceder al hardware directamente, asegurando operaciones controladas y seguras
* Asegura la multitarea, gestión de recursos y protección, haciendo a UNIX estable, eficiente y seguro

Kernel y su diagrama de bloque

El kernel y su diagrama de bloque ilustra como el kernel de UNIX gestiona system calls, procesos, archivos e interacciones con el hardware en una forma estructurada

https://media.geeksforgeeks.org/wp-content/uploads/20260122104404160306/system_call_interface.webp

* Nivel 1. Usuario: Esta capa contiene programas de usuario y aplicaciones, quienes solicitan servicios  del sistema operativo usando system calls. Los system calls se comportan como funciones normales de C, accedidas a través de bibliotecas en tiempo de compilación

System call e interfaz de biblioteca
- Funciona como la frontera de comunicación entre programas de usuario y el kernel, asegurando un acceso seguro al sistema
- Programas de C usan funciones de biblioteca,  mientras que los programas de ensamblador pueden invocar system calls directamente

* Nivel 2. Nivel de kernel: El nivel de kernel gestiona operaciones de sistema principales (core) como procesos, archivos, memoria y control de dispositivos interactuando directamente con el hardware

1. Subsistema de archivos
- Maneja tareas relacionadas a archivos como asignación (allocation), permisos de acceso, recuperación de datos y almacenamiento
- Provee system calls relacionados a archivos como open, read, write, close, stat, chmod y chown

2. Buffering y bloque entrada/salida (block I/O)
- Regula el flujo de datos entre el kernel y los dispositivos de almacenamiento a través de un mecanismo de buffering
- Funciona con controladores de dispositivos block I/O para gestionar eficientemente operaciones de lectura y escritura en disco

3. Controladores de dispositivos
- Módulos del kernel que controlan componentes de hardware como discos, impresoras y terminales
- Soporta dispositivos buffered de bloqueo y dispositivos puros que sobrepasan el buffering

4. Subsistema de control de procesos
- Gestiona creación, planeamiento, ejecución y terminación de procesos
- Trabaja de cerca con el subsistema de archivos y asigna tiempo de CPU a procesos en ejecución

* Nivel 3. Nivel de hardware: consiste en componentes físicos tales como el CPU, memoria y dispositivos de entrada/salida; genera "interrupts" que el kernel debe manejar meintras gestiona tareas en curso
