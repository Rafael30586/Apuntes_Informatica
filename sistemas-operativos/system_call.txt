Los programas de usuario no pueden acceder directamente al hardware o recursos críticos del sistema operativo porque haría que el sistema se vuelva inestable e inseguro. Para mantener la seguridad, el sistema operativo provee system calls: interfaces controladas que permiten a los programas de usuario solicitar servicios del kernel. Estas llamadas funcionan como puertas de entrada entre el modo usuario y el modo kernel. Los system calls son:
* Una forma que tienen los programas de usuario de interactuar con el sistema operativo;
* Proveen los servicios del sistema operativo a los programas de usuario;
* Solo puntos de entrada en el kernel son ejecutados en modo kernel.

Ejemplos:
* Abrir un archivo en C (fopen) usa internamente system calls como open()
* Ejecutar un programa en Linux usa los system calls fork() y exec()

żCómo funcionan los system calls?
* El programa de usuario ejecuta una instrucción de system call 
* El CPU cambia de modo usuario a modo kernel para una ejecución segura
* El kernel identifica el número de system call y lleva a cabo la operación solicitada (acceso a archivos, creación de proceso, asignación de memoria, etc)
* Después de completar una tarea, el kernel cambia a modo usuario
* El resultado (éxito, error, datos) es devuelto al programa
* Sin system calls, cada programa necesitaría su propia manera de acceder al hardware, llevando a sistemas inconsistentes e inseguros.

Los system calls no siempre causan un cambio de contexto. Involucran principalmente un cambio de modo de modo usuario a modo kernel. Un cambio de contexto ocurre solo cuando el proceso de llamado es bloqueado, no durante cada llamado de sistema.

Tipos de system calls
Los servicios proveídos por el sistema operativo están típicamente relacionados a cualquier tipo de operación que un programa de usuario puede llevar acabo, como creación, terminación, comunicación, etc. Similares tipos de operaciones son agrupadas en una sola categoría de system call. Los system call se clasifican en las siguientes categorías: 
* File system: Usados para crear, abrir, leer, escribir y gestionar archivos y directorios.
* Proccess control: usados para crear, ejecutar, sincronizar y terminar procesos.
* Gestión de memoria: usados para asignar, liberar y gestionar memoria para procesos.
* Interprocess communication (IPC): usado para intercambio de datos y comunicación entre diferentes procesos.
* Gestión de dispositivos: usados para solicitar y liberar dispositivos, y llevar a cabo operaciones de lectura/escritura en ellos.
