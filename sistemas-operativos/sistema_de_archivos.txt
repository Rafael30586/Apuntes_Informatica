¿Qué es un sistema de archivos?
En computación, un sistema de archivos es un sistema lógico y físico para organizar, gestionar y acceder a los archivos y directorios en un SSD, HDD u otros medios. Sin un sistema de archivos, el sistema operativo vería solamente grandes pedazos de datos sin forma de distinguir un archivo del otro.
Los sistemas de archivos pueden ser bastante diferentes entre distintos sistemas operativos como Windows, macOS y Linux. Por otro lado, un sistema operativo puede soportar múltiples sistemas de archivos a pesar de sus diferencias. 

---------------------
¿Cómo funcionan los sistemas de archivos?
Un sistema de archivos almacena y organiza datos. Puede ser pensado como un índice para todos los datos contenidos en el dispositivos de almacenamiento. En adición a SSDs y HDDs, los sistemas de archivos se usan para discos ópticos, unidades flash y cintas magnéticas.
Los sistemas de archivos especifican convenciones para nombrar archivos, incluyendo la máxima cantidad de caracteres en un nombre, que caracteres pueden ser usados y que tan grande puede ser la extensión del archivo. En muchos sistemas operativos, los nombres de los archivos no son "case sensitive".
Junto con el contenido de un archivo, un sistema de archivos mantiene metadatos sobre los archivos junto con otra información. Los metadatos incluyen detalles como el tamaño del archivo, fecha de creación o ubicación en el directorio.
La mayoría de los sistemas organizan los archivos en una jerarquía de directorios, con la ubicación del archivo descrito por su ruta dentro de la estructura de directorios. Los directorios (carpetas en windows) están organizados en una estructura invertida jerárquica de árbol, con el directorio raíz en la cima. Cada archivo es ubicado en un directorio o subdirectorio a la ubicación deseada dentro de la estructura de árbol.

Antes de que los archivos y directorios puedan ser definidos en un medio de almacenamiento, una o más particiones deben ser creadas. Una partición es una región de almacenamiento que el sistema operativo gestiona separadamente de otras particiones, como gestionando unidades (o discos) individuales. Cada partición puede ser formateada con un sistema de archivos diferente para mayor flexibilidad.
Particiones separadas ayudan a aislar archivos y operaciones entre sí, lo que puede mejorar el rendimiento, seguridad y mantenimiento. Por ejemplo, un SSD de una computadora podría contener tres particiones, una para los archivos del sistema operativo, una para los archivos del usuario y otra para loa archivos de recuperación del sistema. Si el sistema de archivos en una partición se corrompe, los datos en otra partición deberían estar a salvo. 

---------------------------
Arquitectura del sistema de archivos

Los sistemas de archivos se encargan de operaciones lógicas y físicas asociadas a la gestión de archivos. Aunque tienen diferentes estrategias para almacenar los datos de los archivos, comúnmente separan a los metadatos del contenido del archivo. También almacenan una pequeña cantidad de otros tipos de datos, que son usados para mantener el sistema de archivos y facilitar el acceso a archivos.
Un sistema de archivos divide una partición en bloques, y su tamaño depende del sistema de archivos. La mayoría de los bloques se usan para el contenido de los archivos. Los bloques restantes se dividen entre metadatos de archivos y gestión de datos del sistema de archivos. La siguiente imagen provee información sobre como un sistema de archivos en un entorno tipo UNIX podría organizar los datos a través de los bloques individuales. En este caso, los metadatos son almacenados en "inodes" (index nodes). Un inodo es una estructura de datos indexada que mantiene detalles sobre un archivo o directorio.

https://www.techtarget.com/rms/onlineimages/file_systems_architecture-f.png

Los datos de gestión ocupan los primeros bloques, seguidos por los metadatos de inodos y luego los bloques con el contenido, lo que consume la mayor parte del almacenamiento. Acá hay un desglose de los diferentes tipos de bloques:
* Super block: este bloque contiene información sobre el sistema de archivos propiamente dicho, funcionando como su propio repositorio de metadatos. Por ejemplo, el superblock podría incluir detalles sobre el tipo de sistema de archivos, número total de bloques, número de inodos u otros componentes.
* Data block bitmap: este bloque contiene un simple mapa de bits que indica que data blocks están usados o en desuso. El valor 1 indica que el bloque está en uso, y el 0 indica desuso.
* Inode bitmap: el bloque contiene un simple mapa de bits que indica que inodos están usados o en desuso, utilizando 1s y 0s de manera similar al item anterior.
* Inodos: el sistema de archivos mantiene un inodo por cada archivo y directorio. El inodo es identificado por un número único e incluye los metadatos para el archivo asociado o el directorio. También incluye la dirección de los data blocks que contienen el contenido del archivo. En este sentido, el inodo puede utilizarse para localizar el archivo rápidamente.
* Content data blocks: todo el contenido de los archivos es almacenado en los data blocks. Estos bloques representan a la mayoría de los bloques ocupados del archivo. La imagen anterior solo muestra una porción de los content data blocks.

Los sistemas de archivos usan metadatos para almacenar y recuperar archivos y para proveer a los usuarios con información sobre sus archivos. Los sistemas de archivos no necesariamente rastrean los mismos tipos de metadatos, pero generalmente mantienen la siguiente información:
* Fecha y hora de creación
* Fecha y hora de modificación
* Fecha y hora de último acceso
* Dueño del archivo
* Permisos de acceso
* Tamaño del archivo, incluyendo tamaño en disco
* Atributos como solo lectura u oculto
* Ubicación en la jerarquía de directorios

Los usuarios pueden típicamente acceder a los metadatos de los archivos a través de la interface del sistema. En Windows, por ejemplo,  se puede hacer click derecho e ir a propiedades en el menú.

----------------------
Acceso al sistema de archivos

Los sistemas de archivos pueden restringir la lectura y escritura a un usuario en particular o un grupo de usuarios.
Permisos de directorios y archivos pueden ser usados para controlar quien tiene acceso a recursos específicos y que nivel de acceso tienen. Esta estrategia es útil para prevenir acceso para usuarios internos pero no es tan efectivo contra usuarios de afuera.
La encriptación de archivos puede también prevenir el acceso de usuarios, pero está enfocado en proteger a los sistemas de ataques externos. Una clave de encriptado es aplicada a un texto sin encriptar para encriptarlo. La clave puede ser usada también para desencriptar el texto encriptado. Solo usuarios que tengan la clave pueden desencriptar el archivo. Con la encriptación, el sistema de archivos no necesita saber la clave de encriptado para gestionar los datos eficientemente.

Tipos de sistemas de archivos

La mayoría de los sistemas operativos soportan múltiples sistemas de archivos, todos con estructuras lógicas y físicas variadas. Algunos sistemas de archivos pueden usarse a través de múltiples plataformas. Los sistemas de archivos principales usados en plataformas como Windows, Linux, macOS o Android son:
* File allocation table (FAT): es un sistema de archivos simple y confiable que solía usarse mucho en viejas versiones de Windows. Diseñado en 1977 para disquetes, fue luego adaptado para discos duros. Originalmente FAT era un sistema de 8 bits, pero luego fue actualizado a FAT12 (12 bits), luego FAT16 (16 bits) y finalmente FAT32 (32 bits), que es la versión que está todavía en uso. Aunque es eficiente y compatible con la mayoría de los SOs actuales, no puede competir con el rendimiento y escalabilidad de los sistemas más modernos.
* Extended file allocation table (exFAT): es el sucesor de FAT32. Conserva bastante la simplicidad de FAT y su fácil implementación. Pero exFAT es un sistema de 64 bits, por lo que puede soportar dispositivos de almacenamiento de mayor capacidad así como aplicaciones que dependen de archivos más grandes. Este sistema también incorpora extensibilidad en su diseño, haciendo que sea más fácil adaptarse a cambios en almacenamiento y uso.
* New technology file system (NTFS): ha sido el sistema de archivos por defecto de Windows desde Windows NT 3.1. Ofrece varias mejoras en comparación con FAT, incluyendo mejor rendimiento, soporte para metadatos y utilización de recursos. NTFS es también soportado en Linux a través de un driver de código abierto. Además, macOS incluye soporte de solo lectura pata NTFS.
* Extended file system (ext): implementado en 1992, este fue diseñado específicamente para Linux y se usa bastante hoy en día. La versión actual, ext4 construida sobre ext3, que añadió  algunas capacidades para reducir la corrupción de datos. La versión ext4 ofrece mejor rendimiento y fiabilidad y soportando mayor escalabilidad. Es el sistema por defecto de múltiples distribuciones de Linux, incluyendo Ubuntu y Debian, y es el principal usado en dispositivos Android.
* B-tree file system (Btrfs): también conocido como butter FS o  better FS, combina un sistema de archivos y un gestor de archivos en una sola solución para sistemas Linux. La solución ofrece tolerancia a fallos avanzada y capacidades de autocuración, resultando en mayor fiabilidad. Es también conocido por su eficiencia y facilidad de administración. Ahora es el sistema por defecto en Fedora Workstation.
* Global File System (GFS): es un sistema de archivos de Linux y también de disco compartido. Ofrece acceso directo al block storage compartido y puede ser usado como un sistema de archivos local. GFS2 es una versión actualizada con características no incluidas en el GFS original, como, por ejemplo, un sistema actualizado de metadatos. Ambos (GFS y GFS2) están disponibles como software libre.
