¿Qué es la memoria virtual?

Memoria virtual es una técnica de gestión de memoria en la cual la memoria secundaria puede ser usada como si fuera parte de la memoria principal. Es una técnica común usada en el sistema operativo de una computadora por lo efectiva que es en la optimización del uso de memoria. 
La memoria virtual usa tanto hardware como software para permitirle a una computadora compensar por falta de memoria física, transfiriendo temporalmente datos de la RAM al almacenamiento en disco. Mapeando pedazos de memoria a archivos de disco permite a la computadora tratar a la memoria secundaria como si fuera la memoria principal.
Hoy en día, la mayoría de las PCs vienen con 8GB de RAM, pero a veces, eso no es suficiente para correr varios programas a la vez. La memoria virtual libera RAM al cambiar datos que no han sido usados recientemente a un dispositivo de almacenamiento, como disco duro o SSD.
La memoria virtual es importante para mejorar el rendimiento del sistema, hacer multitareas y usar programas grandes. De todas formas, los usuarios no deberían depender demasiado de la memoria virtual, ya que es bastante más lenta que la RAM. Si el SO tiene que cambiar datos entre memoria virtual y RAM muy seguido, la computadora empieza a ralentizarse, resultando en una condición llamada thrashing.
La memoria virtual fue desarrollada en una época en la cual la memoria RAM física era cara. Las computadoras tiene  una cantidad finita de RAM, por lo que la memoria se acaba cuando múltiples programas corren simultáneamente. Un sistema usando memoria virtual usa una sección del disco duro o SSD para emular RAM. Con la memoria virtual, un sistema puede cargar programas más grandes o múltiples programas al mismo tiempo, permitiendo a cada uno operar como si tuviera más espacio asignable de disco, sin tener que comprar más RAM. 

------------------------
Como la memoria virtual realmente funciona

La memoria virtual usa hardware y software para operar. Cuando una aplicación está en uso, datos de ese programa se almacenan en una dirección física usando RAM. Una unidad de gestión de memoria (MMU) mapea la dirección a RAM y automáticamente traduce direcciones. La MMU puede, por ejemplo, mapear un address space lógico a la correspondiente dirección de memoria física. 
Si el espacio de la RAM se necesita para algo más urgente, los datos pueden cambiarse fuera de la RAM a la memoria virtual. El gestor de memoria de la computadora está a cargo de rastrear los cambios entre memoria física y virtual. Si los datos son necesitados de vuelta, el MMU de la computadora usa un context switch para reanudar la ejecución.
Mientras se copia la memoria virtual en la memoria física, el sistema operativo divide la memoria con un número fijo de direcciones en pagefiles o swapfiles. Cada página está almacenada en un disco, y cuando la página se necesita, el sistema operativo la copia desde el disco a la memoria principal y traduce las direcciones virtuales en direcciones verdaderas.
De todas formas, el proceso de intercambiar memoria virtual a física es más bien lento. Esto quiere decir que usar memoria virtual generalmente causa una notable reducción en el rendimiento. Gracias al intercambio, las computadoras con más RAM tiene mejor rendimiento.

----------------------------
Tipos de memoria virtual

La MMU de una computadora gestiona las operaciones de memoria virtual. En la mayoría de las computadoras, el hardware MMU está integrado en la CPU. El CPU también genera el address space virtual. En general, la memoria virtual está paginada (paged) o segmentada.
La paginación divide la memoria en secciones o archivos de paginación (paging files).Cuando una computadora agota su RAM disponible, las páginas que no están en uso son transferidas al disco duro o SSD usando un swap file. Un swap file es un espacio reservado en el disco duro para ser usado como la extensión de la memoria virtual para la RAM de la computadora. Cuando el swap file es necesitado, es enviado de vuelta a la RAM usando un proceso llamado "page swapping". Este sistema asegura que al sistema operativo y aplicaciones de la computadora no se les acabe la memoria real. El máximo tamaño del pagefile puede ser 1 y medio a 4 veces la memoria física de la computadora. 
El proceso de paginación de la memoria virtual usa page tables, que traducen las direcciones virtuales que el sistema operativo y las aplicaciones usan a direcciones físicas que el MMU usa. Entradas en la page table indican si la página está en RAM. Si el sistema operativo o un programa no encuentra lo que busca en RAM, entonces el MMU responde a la referencia de memoria faltante con una excepción de falta de página para hacer que el SO mueva la página de vuelta a la memoria cuando se necesita. Una vez la página está en la RAM, su dirección virtual aparece en la page table.
Segmentación también es usada para gestionar memoria virtual. Este enfoque divide la memoria virtual en segmentos de diferentes longitudes. Los segmentos que no están en uso en memoria pueden ser movidos a espacio de memoria virtual en la unidad. Información segmentada o procesos son rastreados en una segment table, lo que muestra si un segmento está presente en memoria, si ha sido modificado y su dirección física. Además, los sistemas de archivos en segmentación están hechos de segmentos que son mapeados a un potencial address space de un proceso. 
La segmentación y la paginación se diferencian, como modelos de memoria, en la forma en que la memoria se divide; de todas formas, los procesos pueden ser también combinados. Cuando se combinan, la memoria se divide en marcos (frames) o páginas. Los segmentos ocupan múltiples páginas, y la dirección virtual incluye tanto el número de segmento como el número de página.
Otros métodos de reemplazo de páginas incluyen "first in, first out" (FIFO), algoritmo óptimo y "least recently used (LRU) page replacement":
* El método FIFO hace que la memoria seleccione el reemplazo para una página que ha estado en la dirección virtual por la mayor cantidad de tiempo
* El método del algoritmo óptimo selecciona reemplazos de página basado en qué página es poco probable que sea reemplazada después de la mayor cantidad de tiempo; aunque difícil de implementar, esto lleva a menos fallos de página
* El método de reemplazo de página LRU reemplaza la página que no ha sido usada por la mayor cantidad de tiempo en la memoria principal
