Gestión de memoria en un sistema operativo

La gestión de memoria es el proceso de controlar y organizar la memoria de la computadora asignando porciones, llamadas bloques (blocks), a diferentes programas ejecutándose para aumentar el rendimiento general del sistema
* La función más importante de un sistema operativo es gestionar la memoria principal
* Soporta múltiples procesos simultáneamente en memoria
* Protege a los procesos de acceso no autorizado
* Permite el intercambio y memoria virtual eficientemente

Técnicas en asignación de memoria 

Usadas por el sistema operativo para asignar, utilizar, y gestionar recursos de memoria para procesos. Varias técnicas ayudan al sistema operativo a gestionar la memoria eficientemente. Pueden ser categorizadas de la siguiente manera:

Intercambio (swapping)
Swapping es una técnica de gestión de memoria en la cual los procesos son movidos temporalmente entre la memoria principal y la secundaria para liberar memoria para otros procesos
* Permite a múltiples procesos ejecutarse eficientemente
* Procesos de más baja prioridad pueden ser intercambiados por otros de mayor prioridad
* Los procesos intercambiados reanudan cuando son cargados nuevamente en memoria
* El tiempo de transferencia depende de la cantidad de datos movidos

Asignación contigua de memoria (Contiguous memory allocation)

Cada proceso es asignado a un solo bloque de memoria contiguo. Todas las instrucciones y datos de un proceso son almacenados en lugares de memoria adyacentes

* Single Contiguous Memory Allocation: es la forma más simple de gestión de memoria. En esta técnica, la memoria principal se divide en dos partes:
  - Una parte es reservada para el sistema operativo
  - La parte restante es asignada a un solo proceso de usuario
Características:
  - Solo un proceso de usuario puede residir en memoria a la vez
  - El sistema operativo ocupa una porción fija de memoria
  - No es posible la multiprogramación
  - Simple de implementar y gestionar
Ventajas:
  - Gestión de memoria simple
  - No hay problemas de fragmentación
Desventajas:
  - Pobre utilización de memoria
  - No soporta multitareas o multiprogramación

* Partitioned Memory Allocation: La memoria principal se divide en múltiples particiones contiguas, y cada partición puede retener un proceso. Esta técnica soporta multiprogramación
Partitioned memory allocation se clasigica en:
  - Fixed Partition Allocation 
    $ La memoria se divide en un número fijo de particiones
    $ Cada partición tiene un tamaño fijo
    $ Cada partición puede guardar solo un proceso
    $ Lleva a fragmentación interna
    $ Una vez las particiones están definidas, el sistema operativo rastrea del estado de las particiones de memoria, esto se hace a través de una estructura de datos llamada partition table

Primera dirección de la partición | Tamaño de la partición | Estado
0K                                | 200k                   | asignado
200K                              | 100k                   | libre
300K                              | 150k                   | libre
450K                              | 250k                   | asignado

* Variable partition allocation
  - La memoria se divide en particiones dinámicamente basándose en el tamaño del proceso
  - Reduce la fragmentación interna
  - Sufre de fragmentación externa
Ventajas:
  - soporta multiprogramación
  - mejor utilización de memoria comparado con single contiguous allocation
Desventajas: 
  - problemas de fragmentación
  - gestión de memoria compleja comparado con single continuous allocation

Asignación no-contigua de memoria (Non-contiguous memory allocation)

Es una técnica de gestión de memoria en la cual un proceso se divide en partes más pequeñas y estas partes se almacenan en diferentes ubicaciones no-adyacentes en la memoria principal. A diferencia de la asignación contigua, el proceso entero no necesita ser colocado en un único bloque contiguo de memoria.
Esta técnica es ampliamente utilizada en sistemas operativos modernos porque mejora la utilización de memoria y reduce los problemas de fragmentación

* Características de la asignación no-contigua de memoria
  - un proceso puede ser almacenado en múltiples ubicaciones de memoria
  - mejora la utilización de la memoria disponible
  - reduce la fragmentación externa
  - requiere traducción de direcciones usando hardware de soporte (MMU)
* Ventajas
  - mejor utilización de memoria
  - soporta programas grandes
  - elimina la necesidad de memoria libre contigua
* Desventajas
  - más compleja que la asignación contigua
  - sobrecarga adicional para traducción de direcciones
  - requiere memoria extra para tablas (page table / segment table)
* Técnicas usadas en asignación no-contigua de memoria
  1. Paginación (paging): divide un proceso en páginas de tamaño fijo y la memoria en marcos del mismo tamaño
  2. Segmentación: divide un proceso en segmentos lógicos de tamaño variable como código, datos y stack
  3. Segmentación con paginación: combina segmentación lógica con paginación para reducir fragmentación

Mecanismos de gestión de memoria

Memoria virtual
* Deja correr a un programa incluso si es más grande que la RAM física
* Usa el disco como una extensión de la memoria principal

Algoritmos de reemplazo de página (PRA)
* PRA decide que página remover de la memoria cuando está llena
* FIFO: remueve la página que vino primero
* LRU: remueve la página que menos ase ha utilizado recientemente
* Optimal: remueve la página no necesitada por el tiempo futuro más extenso
* LFU: remueve la página usada con menos frecuencia 

Demand paging
* Carga solamente las páginas que un proceso necesita en memoria
* Reduce el uso innecesario de memoria y I/O
