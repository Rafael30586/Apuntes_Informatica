¿Qué es una page table?

Una page table es una estructura de datos dentro del sistema operativo que mapea direcciones lógicas y direcciones físicas. Almacena otra información sobre cada página, como si está en memoria o en disco, si es de solo lectura o de lectura y escritura, y así sucesivamente. Esta información ayudará al sistema operativo a encontrar fácilmente la ubicación de una página en la memoria principal cuando se la necesite. La siguiente imagen muestra la estructura y entradas de una page table simple: 

https://www.tutorialspoint.com/operating_system/images/page_table.jpg

El propósito principal de una page table es el de rastrear el mapeo entre direcciones lógicas usadas por un programa y las direcciones físicas en la memoria de la computadora. Cuando un programa trata de acceder a una ubicación de memoria en la RAM, el CPU primero revisa la page table para ver el mapeo almacenado para la correspondiente dirección lógica. Si la página ya está en memoria, el CPU puede acceder directamente a ella.
Una falla de página ocurrirá si la página no está en memoria. En tales casos, el sistema operativo necesita cargar la página del disco a la memoria y luego actualizar la page table

Datos almacenados en page table

La page table almacena información importante sobre cada página en la memoria virtual. Esto ayudará al sistema operativo a gestionar la memoria eficientemente. Lo que sigue son los datos clave almacenados en una page table:

1. Número de marco: el número de marco se usa para identificar la ubicación física de una página en la RAM. Por cada página en la memoria virtual, hay un correspondiente marco (frame) en la memoria principal. La page table almacena el número de marco para cada página, lo que ayuda al sistema operativo a localizar la página en RAM cuando se la necesita.
El número de bits requeridos para cada número de marco puede ser calculado usando la fórmula:

número de marcos = tamaño de la memoria física / tamaño de marco

número de bits por número de marco = log2(número de marcos)

Por ejemplo, si el tamaño de la memoria física es 64 MB y el tamaño de marco es de 4 KB, entonces el número de marcos será 16384. Por lo tanto, el número de bits requeridos por cada número de marco será 14

2. Bit válido/inválido: El bit válido/inválido se usa para indicar si una página está actualmente cargada en la memoria física o no. Si el bit es establecido como válido (1), significa que la página está en memoria y puede ser accedida. Si el bit está como inválido (0), significa que la página no está en memoria. El bit se usa para revisar si el fallo de página ocurrirá o no. Lo que significa que si un programa trata de acceder a una página que está marcada como inválida, entonces el fallo de página ocurrirá

3. Bits de protección: Los bits de protección se usan para describir los permisos de acceso para una página. Estos bits determinan si una página puede ser leída, escrita o ejecutada por un programa. Bits de protección comunes incluyen:
* Lectura (R) - indica que la página puede ser leída
* Escritura (W) - indica que se puede escribir en la página
* Ejecución (X) - indica que la página puede ser ejecutada como código

Estos bits previenen de acceso no autorizado de páginas de memoria por diferentes procesos. Por ejemplo, un página que contenga código no debería ser escribible, y una página conteniendo datos no debería ser ejecutable

4. Bit de referencia: el bit de referencia se usa por algoritmos de reemplazo de página para rastrear si una página ha sido accedida recientemente. Cuando una página es accedida, el bit de referencia se establece en 1. Esta información ayuda al sistema operativo y los algoritmos de reemplazo de página a tomar decisiones sobre que páginas reemplazar cuando un error de página ocurre. Las páginas que han sido accedidas recientemente (con el bit de referencia en 1) son menos propensas a ser reemplazadas comparadas con las páginas que no han sido accedidas recientemente (con el bit de referencia en 0)

5. Bit de chaché: El bit de caché se usa para indicar si una página puede ser cacheada en la caché del CPU o no
* Si el bit de caché está en 1 --> la página puede ser cacheada
* Si el bit de caché está en 0 --> la página no puede ser cacheada

Este bit es importante para optimización de rendimiento. Las páginas que son accedidas frecuentemente  

6. Bit sucio (dirty bit): el bit sucio se usa para indicar si una página ha sido modificada desde que fue cargada en memoria
* Bit sucio está en 1 --> la página ha sido modificada
* Bit sucio está en 0 --> la página no ha sido modificada

Esta información es importante para algoritmos de reemplazo de página. Si la página con el bit sucio establecido en 1 es seleccionada para reemplazo, debe ser escrita de vuelta en el disco antes de ser reemplazada. Esto asegura que ningún cambio hecho en la página se pierda

Tipos de page tables

Dependiendo de la arquitectura y requerimientos del sistema, hay varios tipos de page tables usadas en sistemas operativos. Algunos de los tipos comunes incluyen:

1. Page table de un solo nivel (single level page table): en una page table de un solo nivel, hay una sola tabla que mapea todas las páginas de un proceso a sus correspondientes marcos en la memoria física. Este tipo de page table es fácil de implementar. Pero puede no ser conveniente para sistemas con address spaces grandes, porque la page table se puede volver muy grande y consumir un montón de memoria

2. Page table multi-nivel: En una page table multi-nivel, la page table se divide en múltiples niveles. Cada nivel contiene una page table más pequeña que mapea a una porción del address space. Esta estructura jerárquica ayuda a reducir el tamaño de la page table y la hace más manejable. El tipo más común de página multi-nivel es la page table de dos niveles

3. Page table invertida: En una page table invertida, hay una sola page table para todo el sistema. Cada entrada en la tabla corresponde a un marco en la memoria física. La entrada contiene información sobre qué página está actualmente almacenada en ese marco. Además el ID del proceso al que la página pertenece. Este tipo de page table es más eficiente en lo que respecta a memoria, porque reduce la cantidad de memoria necesitada para almacenar page tables para múltiples procesos. Aún así, puede ser más lento para acceder, porque el sistema necesita buscar a través de la tabla para encontrar la página correspondiente para una dirección lógica dada

Conclusión

Una page table almacena información importante sobre la relación entre direcciones lógicas y direcciones físicas en un sistema de memoria virtual. Ayuda al sistema operativo a gestionar la memoria eficientemente uy en manejar fallos de página. Los datos claves almacenados en una page table incluyen: número de marco, bit válido/inválido, bits de protección, bis de referencia, bits de caché y bit sucio. Hay distintos tipos de page tables, tales como de un solo nivel, multi-nivel y page tables invertidas
