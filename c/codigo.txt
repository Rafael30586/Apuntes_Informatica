Supongamos que tenemos unas variables que declaramos así:

int variable = 10;
int *puntero = &variable;

Podemos imprimir los siguientes valores:

printf("%p", puntero) --> imprime la dirección de memoria que almacena el puntero
printf("%d", *puntero) --> imprime el valor al que apunta el puntero, que en este caso sería 10. A este procedimiento se lo llama en inglés "dereference"

--------------------
Si a una puntero que apunta a un array se le suma un número entero (como 2 o 3 por ejemplo), el puntero se moverá dicha cantidad de posiciones. Ejemplo:

int numeros = {30,12,95,61,90,27};
int *puntero = números;
printf("%d",*puntero); // imprime 30
puntero = puntero + 2;
printf("%d",*puntero); // imprime 95

Cuando le sumamos o restamos números enteros a un puntero que apunta a un valor de un array, este puntero se mueve posiciones dentro del array, no importa de qué tipo sea, si el array es de char, de int, de double, etc. En un array de char, el puntero se mueve 1 posición de memoria, en un array de int, el puntero se mueve 4 posiciones de memoria, etc

---------------------
Si tenemos dos punteros, uno que apunta a la posición 1 de un array y otro que apunta a la posición 4 del mismo array, podemos restarlos para que nos de como resultado la distancia en elementos que hay entre un puntero y el otro

---------------------

Podemos crear punteros que apunten hacia otro puntero, por ejemplo:

int numero = 10;
int *puntero1 = &numero;
int **puntero2 = &puntero1;

-----------------------
Existen en C los punteros función o function pointers que, en lugar de apuntar a una variable, apuntan a una función. Esto significa que almacena la dirección de una función, permitiéndote llamar a la función usando un puntero. Ejemplo:

int (*puntero) (int ,int);

En este caso estamos creando un puntero que guarda la dirección de memoria de una función que retorna un entero y tiene dos parámetros enteros. Podemos asignarle una función así:

puntero = sumar;

o

puntero = &sumar;

Siendo "sumar" una función que hemos creado. Para llamar a la función a través del puntero podemos hacer así:

int suma = puntero(12,56);

o

int suma = (*puntero)(12,56);

Nota: El nombre de una función en sí mismo apunta al comienzo de su código en memoria. Eso quiere decir que el nombre de una función actúa como un puntero; declarar un puntero función simplemente te da una variable que puede almacenar esa dirección.
------------------
En C se puede crear arrays de funciones, por ejemplo:

void (*saludos[3])() = {saludo,saludo2,saludo3};

En este ejemplo el array se llama saludos y almacena 3 funciones: saludo, saludo2 y saludo3. Para llamar, por ejemplo, a saludo2 a través del array se hace así:

saludos[1]();

Funciones callback

Son funciones que son pasadas como argumento a otra función. En C, las funciones callback son implementadas usando punteros función. Por ejemplo:

void ejecutarCallback(void (*callback)){
  callback();
}

typedef

La palabra typedef te permite crear un nuevo nombre (como un alias) para un tipo ya existente. Esto puede hacer que declaraciones complejas sean más fáciles de leer, y tu código más fácil de mantener. Por ejemplo, en lugar de siempre escribir float, podemos crear un nuevo tipo llamado "temperatura" para hacer al código más claro. También se puede usar con estructuras, por ejemplo:

typedef struct {
  char brand[30];
  int year;
} Car;

Reasignación de memoria

Si la cantidad de memoria que asignamos con malloc() o calloc() no es suficiente, podemos reasignar memoria para hacerla más grande
La reasignación reserva una cantidad diferente de memoria (generalmente mayor) mientras mantiene los datos almacenados en ella
Podés cambiar el tamaño de memoria asignada con la función realloc()
La función realloc() tiene dos parámetros:
* el primero es un puntero a la memoria que se quiere redimensionar
* el segundo especifica el nuevo tamaño de la memoria asignada, medido en bytes

Ejemplo:
int *ptr2 = realloc(ptr1, size);

La función realloc() trata de redimensionar la memoria en ptr1 y retornar la misma dirección de memoria. Si no puede redimensionar la memoria en la dirección actual entonces asignará memoria en una dirección diferente y devolverá la nueva dirección en su lugar

Nota: Cuando realloc() retorna una dirección de memoria diferente, la memoria en la dirección original ya no está reservada y no es seguro usarla. Cuando la reasignación está hecha es bueno asignar el nuevo puntero a la variable previa para que el viejo puntero ya no pueda ser usado por accidente

La función realloc() retorna un puntero NULL si no es capaz de asignar más memoria. Esto es poco probable, pero es bueno tenerlo en mente cuando necesitás que tu código sea a prueba de fallos

Ejemplo de como se revisa un posible puntero NULL:

int *ptr1, *ptr2;

// Allocate memory
ptr1 = malloc(4);

// Attempt to resize the memory
ptr2 = realloc(ptr1, 8);

// Check whether realloc is able to resize the memory or not
if (ptr2 == NULL) {
  // If reallocation fails
  printf("Failed. Unable to resize memory");
} else {
  // If reallocation is successful
  printf("Success. 8 bytes reallocated at address %p \n", ptr2);
  ptr1 = ptr2;  // Update ptr1 to point to the newly allocated memory
}

Liberación de memoria

Cuando ya no necesitás un bloque de memoria deberías liberarlo. 
La memoria dinámica permanece reservada hasta que es liberada o hasta que el programa termina
Una vez la memoria es liberada puede ser usada por otros programas o incluso podrá ser asignada a otra parte de tu programa
Para liberar memoria se usa la función free(), la cual tiene un puntero como parámetro

int *ptr;
ptr = malloc(sizeof(*ptr));

free(ptr);
ptr = NULL

Nota: Se considera buena práctica establecer un puntero en NULL después de liberar la memoria para que no la vuelvas a usar por accidente
Si continuás usando memoria después de que fue liberada podrías corromper datos de otros programas o incluso otra parte de tu propio programa
