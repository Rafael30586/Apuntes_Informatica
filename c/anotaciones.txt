Puntero: es una variable que almacena la dirección de memoria de otra variable

En c, el nombre de un array es un puntero al primer elemento del array

------------------------------
Uniones o unions
 
Son muy parecidas a las estructuras en el sentido de que pueden almacenar miembros de diferentes datos, pero hay algunas diferencias:
* E una estructura, cada miembro tiene si propia memoria
* En una unión, todos los miembros comparten la misma memoria, lo que significa que podés utilizar uno de los valores a la vez

-------------------------
Cuando creamos estructuras en C, el compilador podrá añadir bytes extra de padding entre los miembros
Esto se hace para hacer que el programa ejecute más rápido en la computadora, porque la mayoría de los CPUs leen datos más eficientemente si están alineados apropiadamente en memoria
Rara vez debemos preocuparnos por esto a menos que trabajemos con memoria de bajo nivel o formateo de archivos
Padding significa que el compilador a veces añade espacios vacíos dentro de la estructura para mayor velocidad

Veamos la siguiente estructura simple:

struct Ejemplo{
  char a; // 1 byte
  int b;  // 4 byte
  char c; // 1 byte
}

int main(){
  prinf("Tamaño de la estructura: %zu bytes\n", sizeof(struc Ejemplo));
}

Esperarías a que el tamaño sea 1 + 4 + 1 = 6 bytes , pero imprimirá 12 bytes. ¿Por qué? el compilador añade bytes de padding para que el miembro "b" empiece en una dirección de memoria que sea múltiplo de 4. Esto ayuda al CPU a leer más rápido. Así es como la memoira está asignada:

Miembro  |  Bytes  |  Notas
a           1       alamacenado primero
padding     3       añadido para que b empiece en un múltiplo de 4
b           4       alineado a límite de 4 bytes
c           1       siguiente en alamcenarse
padding     3       añadido para hacer que el tamaño total sea múltiplo de 4

¿Cómo reducir el padding? 
El padding depende del orden de los miembros en la estructura. Si agrupás tipos más grandes primero, podés hacer que la estructura sea más chica:

struct Ejemplo{
  int b; // 4 bytes
  char a; // 1 byte
  char c; // 1 byte
}

En este caso la estructura es solo de 8 bytes en lugar de 12, porque se necesita menos padding

¿Por qué es importante?
* El padding ayuda a que los programas corran más rápido al mantener los datos alineados en memoria 
* Puede hacer que la estructura sea más grande de lo esperado
* Reordenar los miembros puede reducir el tamaño total
* Nota: rara vez necesitás preocuparte por esto a menos que trabajes con memoria de bajo nivel o formateo de archivos

Asignar memoria (allocate memory)

El proceso de reservar memoria es llamado asignación (allocation). Al manera de asignar memoria depende del tipo de memoria
C tiene dos tipos de memoria: memoria estática y memoria dinámica

1. Memoria estática
La memoria estática se reserva para variables antes de que el programa corra. La asignación de memoria estática se conoce también como asignación de memoria en tiempo de compilación. 
C automáticamente asigna memoria para cada variable cuando el programa es compilado. Por ejemplo, si creás un array de números enteros de 20 estudiantes, C reservará espacio para 20 elementos, lo cual es típicamente 80 bytes. Pero puede ser que la cantidad de estudiantes sea menos de la esperada, por ejemplo, solo 12 estudiantes, entonces se ha malgastado el espacio de 8 elementos. Como no sos capaz de cambiar el tamaño del array, te quedás con una cantidad innecesaria de espacio reservado. Para tener mejor control de memoria asignada, se puede usar memoria dinámica

2. Memoria dinámica
La memoria dinámica es memoria que es asignada después de que el programa empieza su ejecución.	La asignación de memoria dinámica puede ser también nombrada como asignación de memoria en tiempo de ejecución.
A diferencia de la memoria estática, tenés control total sobre cuanta memoria es usada a la vez. Podés escribir código para determinar cuanta memoria necesitás y asignarla
La memoria dinámica no pertenece a una variable, puede ser accedida solamente con punteros
Para asignar memoria dinámica, podés usar las funciones malloc() o calloc(). Es necesario incluir la cabecera <stdlib.h> para usarlas. Estas funciones asignan algo de memoria y retornan un puntero a su dirección
La función malloc() tiene un parámetro, tamaño, que especifica cuanta memoria asignar, medida en bytes
La función calloc() tiene dos parámetros: cantidad (amount), que especifica la cantidad de ítems para asignar; tamaño (size), que especifica el tamaño de cada item medido en bytes
